# 03 · 画像与场景 · course / **admin**

> 角色定义在 [G3-01](../../../../grules/G3-权限与认证规范/01-角色定义.md);admin 端 3 角色:`readonly` / `content_admin` / `super`。

## 3.1 主画像

### 内容运营(`content_admin`)
| 维度 | 设定 |
|------|------|
| 角色 | 内容编辑团队;1-3 人 / 主题 |
| 工作设备 | Windows / macOS 笔记本 + 27 寸外接显示器 |
| 工作量 | 每日点检 50-200 题;每周发布 1-2 章 |
| `tracks_scope` | 被 `super` 分配;通常 1-2 个主题 |
| 主要任务 | KP / 题目导入 → 点检 → 发布;举报审核 |

### 超管(`super`)
| 维度 | 设定 |
|------|------|
| 角色 | 项目核心成员;1-2 人 |
| 权限 | 全权 + 高危(unpublish > 30 天 / 邀请管理员 / 重置密码) |
| 主要任务 | 管理员邀请;`tracks_scope` 分配;长期撤回审批 |

### 只读(`readonly`)
| 维度 | 设定 |
|------|------|
| 角色 | 项目顾问 / 外部审计 |
| 权限 | 只看 / 不写;所有 P 页面只读;不能下载 |
