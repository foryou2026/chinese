## 0. 摘要

本项目国际化体系分两大支柱：**UI 文案**（代码中的按钮、提示、标签等）和**数据库内容**（教学资料、题目、文章等）。两者共享语言注册表，但存储机制、管理流程、翻译触发方式完全独立。音频/配音作为可选插件模块，与翻译解耦。

管理后台（admin）界面固定为中文，不做多语言。多语言仅面向用户端（app）。

## 1. 核心设计决策

| 项目 | 决定 | 原因 |
|------|------|------|
| 开发源语言 | 中文（`zh`） | 团队母语，开发简洁，与翻译完全解耦 |
| 枢纽语言 | 英文（`en`） | 国际标准语言，AI 跨语言翻译质量最高 |
| 翻译链 | zh → en（AI）→ 其他语言（AI，从 en 翻译） | 英文作为唯一枢纽语言，任何语言间转换必须以英文为媒介 |
| 默认显示语言 | `zh` | 主要用户群 |
| fallback 语言 | `en` | 翻译缺失时暴露英文 |
| 语言检测 | `localStorage` > `navigator.language` > 默认 `zh` | 用户选择优先 |
| 管理后台语言 | 仅中文 | 管理后台面向内部运营，无需多语言 |
| 语言列表来源 | 数据库 `i18n_languages` 表，管理后台动态配置 | 支持运行时增减语言 |
| 受保护语言 | 中文（系统默认）、英文（枢纽语言） | 均不可停用 |

## 2. 枢纽语言翻译链规则

所有翻译必须以英文（`en`）为枢纽语言。具体规则如下：

### 2.1 源语言为中文（zh）时
1. 系统先查询该条目是否已存在英文翻译
2. 若已存在 → 直接用英文翻译为目标语言
3. 若不存在 → 先将中文翻译为英文，再将英文翻译为目标语言
4. 界面上始终展示「中文原文 ↔ 目标语言翻译」对照，英文中间过程不在 UI 暴露

### 2.2 源语言为英文（en）时
- 直接从英文翻译到目标语言，无需中间步骤

### 2.3 源语言为其他语言时
1. 系统先查询是否已存在英文翻译
2. 若已存在 → 直接用英文翻译为目标语言
3. 若不存在 → 先将源语言翻译为英文，再将英文翻译为目标语言

### 2.4 语言管理约束
- 中文（`zh`）：系统默认语言，不可停用
- 英文（`en`）：枢纽语言，不可停用，种子数据中默认启用
- 其他语言：可自由启用/停用

## 3. 两大支柱

### 3.1 UI 文案（代码级）

| 维度 | 说明 |
|------|------|
| 存储 | `packages/shared-i18n/src/*.ts`，TS 模块 |
| 源文案文件 | `zh.ts`（开发时写中文） |
| 运行时 | `i18next` + `react-i18next` |
| 组件使用 | `t('auth.login')` |
| Key 命名 | `snake_case`，按业务域分组 |
| 翻译方式 | 管理后台触发 AI 翻译，结果同步回代码文件 |
| 管理后台同步 | 代码 → API 同步到 `i18n_ui_entries` 表 → 管理后台展示 |
| 新增流程 | 开发者在 `zh.ts` 加 key → 同步 → 管理后台可见 → AI 翻译 → 发布 |

### 3.2 数据库内容（数据级）

| 维度 | 说明 |
|------|------|
| 存储 | 独立通用翻译表 `i18n_content_translations` |
| 索引键 | `(table_name, record_id, field_name, locale)` |
| 字段控制 | `i18n_field_config` 表定义每个字段是否需要翻译 |
| 源语言 | 每字段独立配置 `source_locale`（支持不同字段不同源语言） |
| 翻译方式 | 管理后台触发 AI 翻译 |
| 模型选择 | 表级/字段级/语言级可独立指定翻译模型 |
| 两种场景 | 替换型（如英文释义→越南语释义）与全文翻译型（如中文文章→英文文章），底层逻辑统一 |

### 3.3 音频/配音（可选插件）

| 维度 | 说明 |
|------|------|
| 定位 | 独立模块，与翻译完全解耦 |
| 存储 | `i18n_audio_files` |
| 控制层级 | 表级默认 + 字段级覆盖（层级导航，同内容翻译结构） |
| 接口 | 多供应商多音色模型，表/字段/语言级可指定不同音色 |
| 音色管理 | 独立菜单，按供应商和语言管理音色配置 |
| 通用性 | 绝大多数项目不需要，按需启用 |

## 5. 翻译状态机

所有翻译条目（UI 文案和 DB 内容）共享统一的状态流转：

```
pending ⇄ translated
```

| 状态 | 含义 |
|------|------|
| `pending` | 待翻译（新增、新启用语言、或源文变更后自动回退） |
| `translated` | 已翻译（AI 翻译完成或管理员手动编辑） |

状态流转规则：
- 新增条目 / 新启用语言 → `pending`
- AI 翻译完成 → `translated`
- 管理员手动编辑已翻译内容 → 保持 `translated`
- 源文变更 → 自动回退为 `pending`

## 8. 响应式与断点

> 三端必做，全宽自适应，禁止两侧留白。

### 8.1 断点

| 断点名 | 宽度 | 目标设备 |
|--------|------|---------|
| `sm` | >= 640px | 大屏手机 |
| `md` | >= 768px | 平板竖屏 |
| `lg` | >= 1024px | 平板横屏/小笔记本 |
| `xl` | >= 1280px | 桌面 |
| `2xl` | >= 1536px | 大桌面 |

### 8.2 布局规则

| 规则 | 说明 |
|------|------|
| 容器 | 不使用 `max-width` 固定容器，内容区域 100% 宽度 |
| 内边距 | 移动端 `px-4`，桌面 `px-6` ~ `px-8` |
| Grid | Tailwind CSS Grid / Flexbox |
| 最小宽度 | 320px |

### 8.3 导航

| 设备 | 导航 |
|------|------|
| 移动端 | 底部 Tab Bar + 汉堡菜单 |
| 平板 | 可折叠侧栏 |
| 桌面 | 顶部导航栏 |
