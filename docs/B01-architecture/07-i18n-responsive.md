## 0. 摘要

本项目国际化体系分两大支柱：**UI 文案**（代码中的按钮、提示、标签等）和**数据库内容**（教学资料、题目、文章等）。两者共享语言注册表，但存储机制、管理流程、翻译触发方式完全独立。音频/配音作为可选插件模块，与翻译解耦。

管理后台（admin）界面固定为中文，不做多语言。多语言仅面向用户端（app）。

## 1. 核心设计决策

| 项目 | 决定 | 原因 |
|------|------|------|
| 开发源语言 | 中文（`zh`） | 团队母语，开发简洁，与翻译完全解耦 |
| 翻译链 | zh → en（AI）→ 其他语言（AI，从 en 翻译） | 英文作为枢纽语言，AI 跨语言翻译质量更高 |
| 默认显示语言 | `zh` | 主要用户群 |
| fallback 语言 | `en` | 翻译缺失时暴露英文 |
| 语言检测 | `localStorage` > `navigator.language` > 默认 `zh` | 用户选择优先 |
| 管理后台语言 | 仅中文 | 管理后台面向内部运营，无需多语言 |
| 语言列表来源 | 数据库 `i18n_languages` 表，管理后台动态配置 | 支持运行时增减语言 |

## 2. 两大支柱

### 2.1 UI 文案（代码级）

| 维度 | 说明 |
|------|------|
| 存储 | `packages/shared-i18n/src/*.ts`，TS 模块 |
| 源文案文件 | `zh.ts`（开发时写中文） |
| 运行时 | `i18next` + `react-i18next` |
| 组件使用 | `t('auth.login')` |
| Key 命名 | `snake_case`，按业务域分组 |
| 翻译方式 | 管理后台触发 AI 翻译，结果同步回代码文件 |
| 管理后台同步 | 代码 → API 同步到 `i18n_ui_entries` 表 → 管理后台展示 |
| 新增流程 | 开发者在 `zh.ts` 加 key → 同步 → 管理后台可见 → AI 翻译 → 发布 |

### 2.2 数据库内容（数据级）

| 维度 | 说明 |
|------|------|
| 存储 | 独立通用翻译表 `i18n_content_translations` |
| 索引键 | `(table_name, record_id, field_name, locale)` |
| 字段控制 | `i18n_field_config` 表定义每个字段是否需要翻译 |
| 源语言 | 每字段独立配置 `source_locale`（支持不同字段不同源语言） |
| 翻译方式 | 管理后台触发 AI 翻译 |
| 两种场景 | 替换型（如英文释义→越南语释义）与全文翻译型（如中文文章→英文文章），底层逻辑统一 |

### 2.3 音频/配音（可选插件）

| 维度 | 说明 |
|------|------|
| 定位 | 独立模块，与翻译完全解耦 |
| 存储 | `i18n_audio_config` + `i18n_audio_files` |
| 控制层级 | 表级默认 + 字段级覆盖 |
| 接口 | 每种语言可配置不同 TTS API |
| 通用性 | 绝大多数项目不需要，按需启用 |

## 3. 技术栈

| 组件 | 选型 | 说明 |
|------|------|------|
| 运行时核心 | `i18next` | 国际化运行时 |
| React 接入 | `react-i18next` | hooks 和组件绑定 |
| 资源格式 | TypeScript 模块 | 支持嵌套和类型约束 |
| 共享资源包 | `packages/shared-i18n` | 集中保存 UI 文案资源 |
| 共享配置包 | `packages/shared-config` | locale 列表从 DB 动态加载 |
| 类型约束 | `ResourceShape = typeof zh` | 其他语言结构必须与 zh 一致 |
| 校验脚本 | `verify-i18n.ts` | 检查 key 完整、值非空、placeholder 一致 |
| AI 翻译 | 可配置接口 | 管理后台配置翻译服务 API |

## 4. 翻译状态机

所有翻译条目（UI 文案和 DB 内容）共享统一的状态流转：

```
pending → translated → reviewed → outdated → pending
```

| 状态 | 含义 |
|------|------|
| `pending` | 待翻译（新增或源文变更后） |
| `translated` | AI 已翻译，未人工审核 |
| `reviewed` | 人工审核通过 |
| `outdated` | 源文已变更，需重新翻译 |

## 5. 管理后台功能概览

| 功能板块 | 说明 |
|---------|------|
| 语言管理 | 从全球主流语言列表中启用/停用语言 |
| UI 文案翻译 | 列出所有代码文案 key，管理翻译状态，触发 AI 翻译 |
| 数据库内容翻译 | 按 数据库→表→字段 层级导航，管理内容翻译 |
| 字段配置 | 表级/字段级控制是否需要翻译，配置源语言 |
| 翻译任务 | 批量/选择性触发翻译，查看进度 |
| 配音管理 | 可选模块，配置 TTS 接口，管理音频文件 |

> 详细需求、交互规范、数据表设计、API 接口定义见 C01-D02 循环下 `i18n/translation-hub` 功能模块。

## 6. App 端集成要点

| 要点 | 说明 |
|------|------|
| 语言切换 | 下拉选择器，选项来自 DB 已启用语言列表 |
| 持久化 | `localStorage` 存储用户语言偏好 |
| 登录前可切换 | 登录页即提供语言切换入口 |
| UI 文案 | `t('key')` 运行时切换 |
| DB 内容 | 前端按当前 locale 从翻译表读取，fallback 到源语言 |

## 7. 响应式与断点

> 三端必做，全宽自适应，禁止两侧留白。

### 7.1 断点

| 断点名 | 宽度 | 目标设备 |
|--------|------|---------|
| `sm` | >= 640px | 大屏手机 |
| `md` | >= 768px | 平板竖屏 |
| `lg` | >= 1024px | 平板横屏/小笔记本 |
| `xl` | >= 1280px | 桌面 |
| `2xl` | >= 1536px | 大桌面 |

### 7.2 布局规则

| 规则 | 说明 |
|------|------|
| 容器 | 不使用 `max-width` 固定容器，内容区域 100% 宽度 |
| 内边距 | 移动端 `px-4`，桌面 `px-6` ~ `px-8` |
| Grid | Tailwind CSS Grid / Flexbox |
| 最小宽度 | 320px |

### 7.3 导航

| 设备 | 导航 |
|------|------|
| 移动端 | 底部 Tab Bar + 汉堡菜单 |
| 平板 | 可折叠侧栏 |
| 桌面 | 顶部导航栏 |

## 8. 格式化

| 项目 | 决定 |
|------|------|
| 时区存储 | UTC (`timestamptz`) |
| 时区展示 | 前端按浏览器时区转换 |
| 货币/数字 | `Intl.NumberFormat` |
| 日期 | `Intl.DateTimeFormat` |
| 相对时间 | `Intl.RelativeTimeFormat` |

## 9. 字体

| 项目 | 决定 |
|------|------|
| 中文 | 系统字体栈：`"PingFang SC", "Hiragino Sans GB", "Microsoft YaHei", sans-serif` |
| 英文/拉丁 | `Inter` 或系统字体 |
| 东南亚语言 | 系统字体栈 |
