# 体验决策原则（7 条）

> **阶段**：B02-XS 体验设计
> **角色**：体验总监 + 设计系统工程师
> **归属**：按系统（app + admin 共享）
> **系统**：app（admin 引用本文件）
> **模块**：全局
> **功能**：全局
> **上游依赖**：01-direction.md
> **冻结状态**：未冻结

---

## P1. 棱光毛玻璃为唯一表面语言

- 表述：所有内容容器（卡片/面板/弹层/下拉/Toast）必须使用 `backdrop-filter` 毛玻璃 + 内顶高光，而非传统实色 + box-shadow。
- 当「实现简易度」与「毛玻璃一致性」冲突时，选毛玻璃一致性。
- 例：Modal 使用 `glass-elevated` + `--glass-blur-lg`，而非白色实色背景。
- 回退：不支持 backdrop-filter 的设备降级为高透明度实色。移动端弱化 blur 至 16px 保性能。

## P2. 弹性灵敏即时反馈

- 表述：用户的每次触控操作必须在 150ms 内产生可感知的视觉变化，使用 spring 缓动。
- 当「精致动画」与「即时响应」冲突时，选即时响应。
- 例：按钮 hover 立即 scale + glow 增强 + 阴影提升，不等待漫长过渡。

## P3. 信息密度优先于呼吸感

- 表述：一屏信息量不能为"留白美学"牺牲。默认表格行高 44px、正文 16px、卡片 padding 20-24px。
- 当「视觉留白」与「信息密度」冲突时，选信息密度。
- 例：首页统计数字始终显示在可视区域，不需要滚动才能看到。

## P4. 五色一键换肤

- 表述：brand 色全局通过 `--color-brand-*` 引用，运行时 `data-accent` 切换 5 套主题色，所有 UI 自动跟随。
- 当「硬编码快速」与「token 间接引用」冲突时，选 token 间接引用。
- 例：按钮 background 必须写 `var(--color-brand-default)`，不可写 `#6366F1`。

## P5. 暗黑模式一等公民

- 表述：暗色模式是独立设计的完整体验（深空黑底、宝石色 mesh、反转中性色、毛玻璃全套覆盖），不是 light mode 的反色补丁。
- 当「快速实现」与「暗色体验完整」冲突时，选暗色体验完整。
- 例：暗色模式 brand 色提亮到 brand-400，毛玻璃基底改为 rgba(255,255,255,0.03~0.07)，阴影加深。

## P6. 文化安全跨境适用

- 表述：颜色、图标、文案在东南亚目标市场（越南、泰国、印尼等）不产生负面文化联想。现代国际化审美优先。
- 当「中国风装饰」与「跨文化安全」冲突时，选跨文化安全。
- 例：中国元素仅在品牌 Logo 处以极简现代方式出现，不在全局 UI 中铺陈。

## P7. 三密可调无障碍

- 表述：通过 `data-density` 支持 default/compact/elder 三档密度，字号/间距/按钮高度通过 token 联动切换。elder 模式字号 19px+、按钮高度 56px。
- 当「固定尺寸简单」与「密度可调」冲突时，选密度可调。
- 例：所有尺寸必须使用 `--space-*` / `--text-*` 变量，不可写死 px。

---

## 优先级排序

```
P1 棱光玻璃 > P2 弹性反馈 > P3 信息密度 > P4 五色换肤 > P5 暗黑一等 > P6 文化安全 > P7 三密可调
```

> P1-P2-P3 构成「视觉体验三角」，是所有决策的底线。P4-P7 构成「灵活性层」，在底线之上追求。
