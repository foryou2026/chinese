# 体验决策原则（7 条）

> **阶段**：B02-XS 体验设计
> **角色**：体验总监 + 设计系统工程师
> **归属**：app（用户学习系统专属）
> **系统**：app
> **模块**：全局
> **功能**：全局
> **上游依赖**：01-direction.md
> **冻结状态**：未冻结

---

## P1. 闯关路径为核心驱动力

- 表述：用户打开 App 看到的不是一个课程列表，而是一条蜿蜒的闯关路径。每个关卡节点有「完成」「进行中」「锁定」三态，形成清晰的学习进度感和目标驱动力。
- 当「快速列表展示」与「路径化呈现」冲突时，选路径化呈现。
- 例：首页主视图为垂直滚动的关卡路径，每个节点 64px 圆形 + 品牌色渐变 / 灰色锁定。

## P2. 毛玻璃增强品质层次感

- 表述：所有浮起容器（导航栏/卡片/弹层/下拉/Toast）使用 `backdrop-filter` 毛玻璃 + 内顶高光，赋予 UI 层次感和现代品质感，区别于传统扁平 App。
- 当「实现简易度」与「毛玻璃一致性」冲突时，选毛玻璃一致性。
- 例：导航栏使用 `glass-bar`，学习卡片使用 `glass-card`，Modal 使用 `glass-elevated`。
- 回退：不支持 backdrop-filter 的设备降级为高透明度实色。移动端弱化 blur 至 16px 保性能。

## P3. 弹跳微交互即时奖励

- 表述：每次用户操作必须有可感知的视觉/触觉反馈。按钮按下弹跳（scale 变化 + spring 缓动），答对庆祝（星星粒子 + XP 飘字），答错震动（红色闪烁 + shake 动效）。
- 当「精致动画」与「即时响应」冲突时，选即时响应。
- 例：按钮 hover 立即 scale + 阴影增强；答对后 500ms 内完成庆祝动画。

## P4. Google 四色一键换肤

- 表述：brand 色全局通过 `--color-brand-*` 引用，运行时 `data-accent` 切换 Google 四原色（红/黄/蓝/绿），所有 UI + 极光点缀自动跟随。
- 当「硬编码快速」与「token 间接引用」冲突时，选 token 间接引用。
- 例：按钮 background 必须写 `var(--color-brand-default)`，不可写 `#EA4335`。

## P5. 移动端第一优先级

- 表述：所有设计从 375px 手机屏幕开始，核心学习流程必须在移动端完整可用。桌面端为扩展适配，不是反过来。
- 当「桌面端展示效果」与「移动端可用性」冲突时，选移动端可用性。
- 例：按钮最小高度 48px（移动端触控友好）、底部导航 Tab 为移动端主导航、表单单列布局。

## P6. 暗黑模式一等公民

- 表述：暗色模式是独立设计的完整体验（深色底、宝石色 mesh、反转中性色、毛玻璃全套覆盖），不是 light mode 的反色补丁。
- 当「快速实现」与「暗色体验完整」冲突时，选暗色体验完整。
- 例：暗色模式 brand 色提亮到 brand-400，毛玻璃基底改为 rgba(255,255,255,0.04~0.08)，阴影加深。

## P7. 文化安全跨境适用

- 表述：颜色、图标、文案在东南亚目标市场（越南、泰国、印尼等）不产生负面文化联想。现代国际化审美优先。
- 当「中国风装饰」与「跨文化安全」冲突时，选跨文化安全。
- 例：中国元素仅在品牌 Logo 处以极简现代方式出现，不在全局 UI 中铺陈。

---

## 优先级排序

```
P1 闯关路径 > P2 毛玻璃品质 > P3 弹跳奖励 > P4 四色换肤 > P5 移动优先 > P6 暗黑一等 > P7 文化安全
```

> P1-P2-P3 构成「游戏沉浸三角」，是所有决策的底线。P4-P7 构成「灵活性层」，在底线之上追求。
