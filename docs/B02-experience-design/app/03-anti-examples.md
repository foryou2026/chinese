## ANTI-1：纯白灰实色 SaaS 模板（典型：飞书、Notion、通用 Admin 后台）

- **典型表象**：纯白 #FFF / 纯灰 #F5F5F5 背景、标准 box-shadow 卡片、无毛玻璃、方正圆角 8px、实线边框分隔、无个性
- **若走偏会怎样**：产品沦为"又一个 SaaS 工具"，用户不会想截图分享，零游戏沉浸感
- **防御机制**：R1（禁止纯白死板背景）；A2（毛玻璃增强层次感）

## ANTI-2：装饰国风过度堆砌（典型：国风手游 UI、政府/旅游网站）

- **典型表象**：卷轴杆、大幅毛笔水墨字背景、满屏楷体/篆体、印章替代所有图标、龙凤祥云边框、暖黄"宣纸"底色
- **若走偏会怎样**：视觉噪音淹没信息，用户感知为"旅游网站"而非现代产品，东南亚年轻用户直接流失
- **防御机制**：R2（禁止国风装饰陷阱）；R3（正文禁用楷体/宋体）

## ANTI-3：过度卡通化 / 幼儿化（典型：某些语言学习 App 的"宝宝风"）

- **典型表象**：大量卡通吉祥物、满屏 emoji、过饱和荧光色、圆滚滚的字体、"宝宝学汉字"的感觉
- **若走偏会怎样**：目标用户是中青年，不是儿童；过度卡通化会让用户觉得"不专业、不高级"
- **防御机制**：色彩饱和度控制在高级感区间；游戏化用数值（+10 XP）而非空洞感叹；毛玻璃质感拉升品质

## ANTI-4：无反馈的静默 UI（典型：传统教育平台 / 在线课程网站）

- **典型表象**：点击按钮无动效、答对答错无视觉差异、没有进度感知、没有成就激励
- **若走偏会怎样**：用户缺乏"游戏感"驱动力，学习变成苦差事，留存率断崖式下降
- **防御机制**：R5（禁止无反馈静默交互）；A4（弹性微交互奖励）；A1（闯关路径驱动）

## ANTI-5：纯桌面端设计下推移动端（典型：传统 Web 优先的学习平台）

- **典型表象**：移动端只是桌面版的缩小版、表格横向滚动、弹窗被截断、按钮太小无法触控
- **若走偏会怎样**：绝大多数语言学习场景在手机上，移动端体验差直接流失核心用户
- **防御机制**：A5（移动端第一优先级）；所有设计从 375px 起始
