# 体验决策原则（5 条）

> **阶段**：B02-XS 体验设计
> **角色**：体验总监 + 设计系统工程师
> **归属**：admin（管理后台专属）
> **系统**：admin
> **模块**：全局
> **功能**：全局
> **上游依赖**：01-direction.md
> **冻结状态**：未冻结

---

## P1. 克制 Frost 为主材质

- 表述：所有容器（卡片/面板/弹层/下拉）使用低饱和毛玻璃（blur ≤ 28px, saturate ≤ 1.5）+ 1px 半透明边框。页面底色 #F3F4F6 + 微弱径向渐变，让毛玻璃层可感知。
- 当「视觉丰富度」与「干净利落」冲突时，选干净利落。
- 例：卡片背景 `--glass-bg-card`（rgba(255,255,255,0.78)）+ `backdrop-filter: var(--glass-blur)`，边框 `--glass-border`。

## P2. 黑白按钮极致简洁

- 表述：Primary 按钮在亮色模式为纯黑底白字（#18181B / #FAFAFA），暗色模式为纯白底黑字（#FAFAFA / #18181B）。Secondary 按钮为白底/深灰底 + 边框。
- 当「品牌色凸显」与「黑白极致」冲突时，选黑白极致。
- 例：所有 Primary 操作按钮使用 `--btn-primary-bg` / `--btn-primary-text`。

## P3. 信息密度优先于留白

- 表述：一屏信息量不能为"留白美学"牺牲。默认表格行高 40px、正文 14px、卡片 padding 16px。
- 当「视觉留白」与「信息密度」冲突时，选信息密度。
- 例：表格默认展示尽可能多的行数，统计数字始终在可视区域。

## P4. 暗黑模式一等公民

- 表述：暗色模式是独立设计的完整体验，基底为 #111827 深灰（非纯黑），卡片为 #1F2937，按钮颜色完全对调。
- 当「快速实现」与「暗色体验完整」冲突时，选暗色体验完整。

## P5. 操作路径最短

- 表述：任何管理操作的点击次数不超过 3 次即可完成。侧边栏一级导航 + 内容区直接操作。
- 当「分步引导」与「直接操作」冲突时，选直接操作。

---

## 优先级排序

```
P1 克制 Frost > P2 黑白按钮 > P3 信息密度 > P4 暗黑一等 > P5 路径最短
```
