## 用途

表单是管理后台的核心交互模式，用于数据录入、配置修改、筛选查询。

## 布局变体

| 变体 | 说明 | 适用场景 |
|------|------|---------|
| standard | 标签在上方，单列 | 创建/编辑表单 |
| inline | 标签在左侧，标签宽度固定 120px | 设置页、详情页编辑 |
| compact | 字段水平排列，适合筛选栏 | 列表页筛选条件 |

## 间距

| 属性 | 值 |
|------|-----|
| 字段间距 | `var(--space-4)` 16px |
| 分组间距 | `var(--space-7)` 32px |
| 标签到输入框 | `var(--space-1)` 4px |
| 操作栏上间距 | `var(--space-6)` 24px |

## 操作栏

- 位于表单底部
- 主按钮靠右：`[取消] [提交]`
- 危险操作单独靠左：`[删除]   ... [取消] [保存]`

## 校验

| 时机 | 行为 |
|------|------|
| 失焦 | 单字段即时校验 |
| 提交 | 全量校验，首个错误字段自动聚焦 |
| 输入中 | 清除当前字段错误态 |

## 加载态

- 提交按钮 loading + disabled
- 表单字段 disabled
- 禁止重复提交

## 反例

- 不使用分步表单的游戏化进度条（admin 使用标准 Stepper）
- 不使用毛玻璃表单卡片

## a11y 验收点

- [ ] `<form>` 包裹

- [ ] 错误消息 `aria-describedby`
