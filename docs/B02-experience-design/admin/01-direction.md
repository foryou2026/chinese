## 一句话

> **「干净、克制、高效。每一像素都服务于操作效率。」**
>
> 它必须**让人信赖**（专业稳重的视觉质感），**让人高效**（信息密度优先，操作路径最短），**让人舒适**（长时间使用不疲劳）。

---

## 5 条不可逾越

| 编号 | 红线 | 可验证标准 |
|------|------|-----------|
| R1 | 禁止花哨装饰 | 无 mesh blob、无装饰性动画、毛玻璃饱和度 ≤ 1.5 |
| R2 | 禁止 hex 硬编码 | 所有颜色必须走 CSS 变量 |
| R3 | 禁止正文使用楷体/宋体 | 全部使用 `--font-sans`（Inter + Noto Sans SC） |
| R4 | 禁止动效 > 300ms | 所有过渡 ≤ 300ms；管理后台追求即时响应 |
| R5 | 亮/暗按钮颜色必须对调 | 亮色模式按钮为纯黑 #18181B；暗色模式按钮为纯白 #FAFAFA |

> 任何 PR / 设计稿命中以上任一条 → 打回，不讨论。

## 5 条主张

| 编号 | 主张 | 可验证维度 | 参考 | 反例 |
|------|------|-----------|------|------|
| A1 | 克制 Frost 材质：页面底色 #F3F4F6 + 微弱径向渐变，卡片/导航/弹层使用低饱和毛玻璃（blur ≤ 28px, saturate ≤ 1.5） | 毛玻璃克制、不花哨、有 `@supports` 回退 | 飞书/Notion | app 的高饱和极光毛玻璃 |
| A2 | 黑白按钮极致简洁：亮色模式 Primary 按钮为纯黑底白字；暗色模式为纯白底黑字 | 所有 Primary 按钮颜色对调 | Notion / Linear | 品牌色渐变按钮 |
| A3 | 侧边栏+顶栏双轴导航：左侧折叠式侧边栏 + 顶部工具栏 | 侧边栏展开 240px / 折叠 48px | 飞书/Notion | 底部 Tab 导航 |
| A4 | 信息密度最大化：默认表格行高 40px、正文 14px、紧凑间距 | 一屏展示尽可能多的数据行 | Linear / 飞书 | 大留白极简 |
| A5 | 数字 tabular-nums：表格/统计/时间戳统一启用 | `font-variant-numeric: tabular-nums` 全局生效 | Linear/Stripe | 数字错位跳动 |

## 检验标准

> 拿新设计稿来，1 分钟内判定通过/不通过。

1. 截图缩到 30%，是否一眼感觉"这是一个专业的后台管理系统"？→ 否则不通过
2. 亮色模式下 Primary 按钮是否为纯黑？→ 否则不通过
3. 暗色模式下 Primary 按钮是否为纯白？→ 否则不通过
4. 毛玻璃饱和度是否超过 1.5？是否有 mesh blob 或装饰性渐变？→ 有则不通过
5. 切换 `data-mode="dark"` 后对比度是否 WCAG AA？→ 否则不通过
6. 侧边栏是否可折叠？→ 否则不通过
7. 页面是否有任何装饰性元素（动画背景、浮动方块等）？→ 有则不通过
