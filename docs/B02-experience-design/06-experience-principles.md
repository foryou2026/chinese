## 1. 4 不原则

### 1.1 不打断

**含义**：能就地内联展示就不弹 Modal；能在当前页面完成就不跳页。

**规则**：
- 编辑型表单优先 **内联 + 浮层（Popover）**；只有"危险操作"才弹 Modal；
- 列表里的"启用 / 禁用 / 修改状态" 走内联 switch + 乐观更新，**不弹确认**（除"删除"）；
- 跳转新页前必须问自己："这一步能否在当前页完成？" 答案是"能"则不跳；
- Toast 不打断操作焦点（不抢焦 / 不挡按钮）。

**反例**：
- ❌ 点击"修改头像" → 弹 Modal → Modal 里再点"上传" → 又弹文件选择 → ...
- ❌ 看课程列表 → 点"详情" → 跳新页 → 看完 → 浏览器返回 → 列表滚动位置丢了

### 1.2 不打扰

**含义**：成功不必告诉用户；只有用户需要知道的才提示。

**规则**：
- **成功的 idle**：保存成功后按钮恢复 idle 即可；如必要再 Toast，但不强制；
- **失败的 Toast**：网络 / 5xx 错误用右上 Toast；不阻塞页面；
- **可恢复错误（4xx / 表单校验）**：内联在字段旁，**绝不**用 Toast；
- 通知中心 / 红点徽标在用户实际进入相关页面后才清；不在登录瞬间清；
- 不主动发"在吗" "今天还没学习"等"类人催促"。

### 1.3 不模糊

**含义**：每个状态有明确反馈，不让用户猜"是不是点了"。

**规则**：
- 按钮点击后立即进入 **loading + disabled** 双重态；
- 异步任务（如生成 / 翻译 / 支付）必须有 **进度文案变化**："正在生成..." → "正在保存..." → "已完成"；
- 任何"做了但你看不到"的副作用必须有反馈（最弱反馈：按钮回 idle）；
- 错误提示必须三件套（是什么 / 为什么 / 怎么办，详见 [04-voice-tone §3](./04-voice-tone.md)）。

### 1.4 不让用户等

**含义**：感知性能优先于真实性能。

**阶梯**：

| 操作耗时 | 反馈 |
|---------|------|
| < 100ms | 不显式反馈（瞬时） |
| 100–300ms | 按钮 disabled，无 spinner（避免闪烁）|
| 300–1000ms | 按钮 disabled + spinner / 骨架屏 |
| 1–5s | 骨架屏 + 进度文案 |
| > 5s | 骨架屏 + 进度文案 + **可取消按钮** |
| > 10s | 移到后台任务，列表占位 + 完成后 Toast |

## 2. 5 个互动公约

### 2.1 直说事

每个按钮 / 提示直接说做了什么，不写"操作成功" / "保存成功"。

- "保存" → 完成时按钮回 idle 或显"已保存"；
- "提交订单" → 完成时跳订单详情而不是 Toast"成功"；
- "重置密码" → 完成时显"密码已重置，请使用新密码登录"。

### 2.2 给反馈

哪怕只是 0.5 秒的微动效，也比沉默好。

- 卡片 hover → 微微"抬起 + 阴影加深"；
- 按钮点击 → 0.1s 内变态；
- 滚动到底 → 显示"已加载全部"或上拉刷新；
- 表单字段 onBlur 立即校验，不等提交。

### 2.3 留草稿

任何需要 ≥ 30 秒填写的表单必须自动留草稿。

- 草稿键 = `<feature>:<form-name>:<userId>`，存 `localStorage`；
- 进入页时自动恢复 + Toast "已恢复上次的草稿"，附"清空"按钮；
- 提交成功后立即清草稿；
- 离开页时表单 dirty → 弹"未保存的更改将丢失"确认（详见 [04 §五](./04-voice-tone.md) + B03）。

### 2.4 可撤销

任何"用户可能后悔"的操作给 5 秒撤销机会。

- 删除消息 / 删除帖子 → Toast"已删除"+ 5 秒倒计时 + "撤销" 按钮；
- 标记完成 / 跳过题目 → 同；
- **真正不可撤销的（删账号 / 退款）** → 走二次确认弹窗 + 输入对象名匹配（[B03 §确认弹窗](../design-system/04-status-colors.md)）。

### 2.5 可重试

网络 / 上游失败必须可重试，且重试不需要重新填表。

- 5xx / 网络 → Toast 含 "重试" 按钮；
- 上游 AI 失败 → 错误卡片含"重新生成"；
- 重试用上次的请求 body，不让用户重填；
- 累计 3 次失败后才升级为"请稍后重试 / 联系客服"。

## 3. 评审清单（贴在 C04 原型 review 模板里）

| 项 | 通过 |
|----|------|
| 是否能就地内联完成？ | ☐ |
| 成功反馈是否最小化？ | ☐ |
| 每个 4 态（idle / loading / success / error）是否都有明确视觉？ | ☐ |
| 错误提示是否三件套？ | ☐ |
| > 300ms 操作是否有骨架 / spinner？ | ☐ |
| > 5s 操作是否可取消？ | ☐ |
| 离开页时表单 dirty 是否有保护？ | ☐ |
| 删除 / 不可撤销操作是否有撤销 / 二次确认？ | ☐ |
| 网络失败是否可重试且不重填？ | ☐ |
| 文案是否避开了禁用词？ | ☐ |
| 是否触犯了任何反例红线（[03](./03-anti-examples.md) §4）？ | ☐ |

## 4. 学习场景的额外公约（汉语学习专属）

1. **学错不羞辱**：任何答错反馈都先承认"难"，再给提示，最后给答案；
2. **学对不夸张**：朴素肯定 "对了" / "做得不错"；不放彩带；
3. **打断学习心流是大罪**：学习页面禁止任何与学习无关的弹窗 / 推送 / 营销；通知必须积压到学习结束后；
4. **节奏感**：题目之间过渡 ≤ 200ms；连续 5 题正确触发"连胜"微反馈；
5. **拼音 / 笔画展示**：用户失误 ≥ 2 次自动给提示，不让用户卡死；
6. **可暂停**：任何学习活动都能暂停且保留进度（包括游戏）。
