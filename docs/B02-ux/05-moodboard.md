# 05 · 情绪板 moodboard

> **阶段**：B03-X  
> **下游**：B04 设计系统选色 / 选纹理 / 选动效曲线时援引  
> **冻结状态**：已冻结 · 2026-04-28

---

## 0. 摘要

- 不是图集，是关键词集合。让"换设计师"也能在同一感受下做事。
- 5 组关键词：情绪 / 色感 / 材质 / 节奏 / 留白。

---

## 1. 情绪关键词（10）

> 任何一屏给人的整体感觉应在这些词的并集中。

- **专注 focused**
- **轻盈 light**
- **沉浸 immersive**
- **现代 modern**
- **温暖 warm**（来自红色 + 暖光晕）
- **不急 unhurried**（动效曲线 + 留白）
- **可信 trustworthy**（克制 + 一致性）
- **稍带俏皮 a touch playful**（仅在成功 / 连胜处）
- **有秩序 ordered**（栅格 + 信息层次）
- **不矫情 unpretentious**（不卖萌 / 不耍宝）

### 反向情绪（出现即打回）

❌ 紧张 / 焦虑 / 杂乱 / 廉价 / 学术 / 教辅 / 儿童 / 工业 / 冷漠 / 喧闹

---

## 2. 色感关键词

> 不写 hex（hex 归 B04）。本文只描述"感觉"。

- 底色：**纸白 / 真黑**——亮模式纸白偏暖（带极淡米色），暗模式真黑（避免纯 `#000`）；
- 强调红：**鲜红偏冷 + 略带粉调**（小红书红，不是中国红 / 不是 Reddit 橙红）；
- 光晕：**极低饱和的红色径向**，像窗外天光，不是霓虹；
- 中性灰：**温灰 warm gray**（带极淡红调），不是冷灰；
- 状态色：**收敛**——成功 / 警告用低饱和绿 / 琥珀；错误复用品牌红。
- 渐变：**禁用业务渐变**；仅远景层、focus ring、按钮 hover 时允许"亮度变化"。

---

## 3. 材质关键词

> 控制毛玻璃在不同位置的表现。

| 位置 | 材质感 |
|------|------|
| 顶栏 | 薄玻璃（中等模糊 + 顶部 1px 高光内边）|
| 卡片 | 中等玻璃（标准模糊 + 高光内边 + 微噪 2.5%）|
| 弹窗 / Drawer | 厚玻璃（强模糊 + 高不透明度 + 较强高光）|
| 按钮（次要）| 薄玻璃 + 小圆角 |
| Toast | 中等玻璃 + 大圆角 + 顶部居中飘入 |
| Tab | 薄玻璃 + Pill 圆角 |

**共同材质细节**：
- 顶部 1px **白色 ~40% alpha 高光内边**（亮模式）/ **白色 6% alpha**（暗模式）—— 灵感源自现实玻璃的反射边；
- 表面**极轻微噪点**（2.5% / 4%）—— 避免"塑料光滑感"；
- 边缘 1px 微弱内边框 —— 让玻璃在白背景上仍"看得见边"。

**反材质**：
- ❌ Material Design 的"层级阴影"过强；
- ❌ 拟物化的 3D 凸起 / 凹陷；
- ❌ 渐变高光（仿苹果按钮）；
- ❌ 完全平面（Flat）。

## 4. 节奏关键词（动效与时间）

| 节奏 | 描述 | 落实 |
|------|------|------|
| **快而稳** quick & stable | hover / focus / 按下 = 极短确认（≤ 120ms）| `--motion-fast` |
| **温柔过渡** soft transition | 卡片切换 / 主交互 ≈ 200ms 缓出 | `--motion-base` |
| **舒展进出** spacious entrance | 弹窗 / Drawer ≈ 320ms ease-out + 透明度 | `--motion-slow` |
| **弹性反馈** elastic feedback | 拖拽 / 列表插入 spring | `--motion-spring` |
| **绝不抢戏** never showy | 任何动效都不超过 1 个屏幕区域 | 评审标准 |

> 全部尊重 `prefers-reduced-motion: reduce` → 退化为 `0ms` 透明度切换。

---

## 5. 留白关键词

- **呼吸而非堆砌**：宁愿留白，不要塞。
- **信息密度**：列表行高桌面 48 / 移动 56；卡片间距 24；区段间距 32。
- **垂直节奏**：标题 + 副标题 + 内容三段必须有明显间距差。
- **横向**：内容区从不居中固定列宽；让视口呼吸（长文段落除外）。

---

## 6. 关键词的使用方式

- B04 设计系统每个 token 的"为什么是这个值" 必须援引本文件一个关键词；
- C04 原型 review 时，必须在每屏 3 秒内能用本文 5 组关键词中至少 3 个描述当前感受；
- 不能描述清楚的屏 → 重做。
