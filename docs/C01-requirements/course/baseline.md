## 0. 摘要

- 给谁：知语 app 端的中文学习者，与 admin 端的内容运营管理员。
- 做什么：以"主题 → 阶段 → 章 → 节 → 知识点 / 题目"为骨架的学习与内容生产闭环；学员侧含学习 / 复习 / 测试 / 个人统计；运营侧含目录 / 节编辑 / 知识点 / 题目 / 举报 / 媒资 / 考试中心 / 全局搜索。
- 最大价值：用"结构稳定 + 内容可发布"的方式把中文学习 SaaS 化，让学员可持续学，运营可批量生产 / 修订。
- 最大风险：发布 / 下架 / 版本治理与"学员历史成绩不追溯"的规则一旦走偏，会导致脏数据或运营反复返工。

## 1. 产品定义与目标

### 1.1 一句话定义

围绕"主题骨架 + 知识点 + 题目 + 测验"做成的中文学习与运营全闭环：学员能按章节循序渐进、按 SRS 间隔复习、按 4 类考试评估；运营能批量生产 / 修订 / 发布课程并实时影响学员可见性。

### 1.2 业务目标

| ID | 目标 | 度量口径 | 验证时点 |
|----|------|--------|---------|
| O1 | 学员首次进入产品后顺利开始第 1 节 | 引导完成率 ≥ 70% | 上线后 4 周 |
| O2 | 学员日活回访 | 7 日留存 ≥ 35% | 上线后 8 周 |
| O3 | 运营内容生产闭环可走通 | 批量导入 + 人工点检 + 发布全程零阻断率 ≥ 95% | 上线后 4 周 |
| O4 | 测验规则正确 | 阶段考通过率与解锁触发率比对 100% 一致 | 持续 |

### 1.3 用户类型

| 类型 ID | 名称 | 画像（≤2 行） | 核心诉求 |
|--------|------|-----------|---------|
| U1 | 学员（app） | 18-35 岁东南亚成年人；母语越 / 泰 / 印尼 / 英 | 用清晰的路径学中文并能持续复习巩固 |
| U2 | 内容运营（admin） | 内部团队，懂中文；负责批量录入与质量把关 | 批量导入 + 行内修订 + 一键发布，不被并发覆盖 |

## 3. 业务场景（叙述）

### 3.1 主要场景

学员入门：零基础学员打开 app，选 UI 语言、注册 / 登录、选目标主题、做自评和短测定级、进入首页，开始第一节学习。

日常学习：学员打开首页看到"继续学"与"今日复习"，先做复习再继续节内学习；遇到错题进入复习队列；答完节末小测拿到结算页。

运营生产：内容运营在管理端看到主题缺口，进入节层级，批量导入知识点 / 题目，预览错误行后导入，行内发布，进入考试中心配置节测蓝图并发布，学员端立即可见。

举报闭环：学员对题目提举报；运营在举报处理页按题目聚合查看，跳转目标编辑面板修订题目，旧作答不重算，新作答按新版。

### 3.2 隐含的重点能力（供 C02-IN 拆解）

- 复习间隔与到期调度
- 发布层级级联与下架边界
- 题目版本治理与"历史成绩不追溯"
- 媒资按内容指纹去重 + 引用拦截
- 学员端"永不读未发布内容"硬约束
- 弱网答题缓冲队列

| 内容编辑 / 发布 / 下架 | — | ✅ |
| 媒资库管理 | — | ✅ |
| 考试中心配置 | — | ✅ |
| 举报处理 | — | ✅ |

### 6.2 数据可见范围

| 角色 ID | 默认可见范围 | 说明 |
|---------|-----------|------|
| ROLE-USER | published-only | 仅已发布内容；自己的成绩 / 进度 |
| ROLE-ADMIN | global（含草稿）| 全部内容含草稿；无学员个人成绩 |

### 6.3 授权校验

引用 `B01-architecture/09-auth-infra.md` 中定义的 `authRequired` / `adminRequired` 中间件。学员端 SRS / 考试 attempt 所有权按 `row.user_id === req.user.id` 判定，不进入角色系统。
