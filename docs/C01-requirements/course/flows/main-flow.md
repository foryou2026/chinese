# `course` · 主流程（业务叙述）

> 流程图与详细状态机详见 [../../../C03-ia/course/](../../../C03-ia/course/)。本文仅以业务语言描述用户场景。

> **覆盖 R-ID**：R-course-001 · 002 · 003 · 004 · 005 · 007 · 008 · 009 · 011 · 013 ~ 023 · 026 · 027

---

## 1. 学员首次进入

1. 学员选择 UI 语言，注册或登录。
2. 学员选择目标主题（5 主题之一）。
3. 学员完成自评与短测定级。
4. 零基础学员被强制进入共享预备主题；其余学员进入所选主题首页。

## 2. 学员日常学习

1. 学员打开首页，看到"继续学"、"今日复习"、"即将解锁"、"推荐"。
2. 学员先做今日复习（按到期 + 间隔规则推送）。
3. 学员继续上次学习的节，从上次最后位置开始。
4. 完成 12 张知识点卡 + 随堂练 + 节末小测 + 结算页。
5. 系统根据答题情况更新复习队列与本节状态。

## 3. 学员考试

1. 学员选择 4 类考试之一：节末小测 / 章测 / 阶段考 / HSK 模考。
2. 进入考试后倒计时启动，不可中途退出（中途退出按零分）。
3. 交卷后看结果；阶段考通过即解锁下一阶段。

## 4. 运营内容生产

1. 运营在 admin 端看到 5 主题缺口统计。
2. 进入主题 / 阶段 / 章 / 节四级列表，选定目标节。
3. 在节编辑面板批量导入知识点 / 题目。
4. 预览错误行 → 修正 → 幂等导入。
5. 在知识点 / 题目筛选行内发布。
6. 在考试中心配置蓝图，试抽预览（不计 attempt）通过后发布。
7. 学员端立即可见。

## 5. 学员举报闭环

1. 学员对题目提举报。
2. 运营在举报处理页按题目聚合查看；累计达阈值的题目自动置顶。
3. 运营跳到目标编辑面板修订题目（题目版本升级）。
4. 旧作答不重算，新作答按新版；处理结果回写举报记录。
