# R01 · 用户输入 · 需求初稿(discover-china)

> **来源**:[`function/01-china/prd/F1-用户-数据与业务规则.md`](../../../../function/01-china/prd/F1-用户-数据与业务规则.md) + [`F2-用户-操作与业务逻辑.md`](../../../../function/01-china/prd/F2-用户-操作与业务逻辑.md) + [`F3-用户-页面与交互描述.md`](../../../../function/01-china/prd/F3-用户-页面与交互描述.md)
> **反向回写期**:2026-05-16

## 一句话定位
"发现中国" Tab,东南亚学员浏览 12 个中国文化类目下的文章 + 逐句中英多语对照 + 中文 TTS 朗读;管理端由超管录入与发布。

## 角色
- **未登录访客**:可看 01-03 类目(中国历史 / 中国美食 / 名胜风光)的列表与详情;
- **登录用户**:可看 01-12 全部类目;
- **super_admin**:管理端 CRUD 类目下文章与句子。

## 数据三元
- **类目** china_categories:12 固定字典,每个有名称 + 介绍,5 语言(中/英/越/泰/印尼);
- **文章** china_articles:挂在类目下,有 12 位编码 + 名称(5 语)+ 拼音 + 状态(待发布 / 已发布);
- **句子** china_sentences:挂在文章下,4 位顺序号,正文 5 语 + 拼音 + 中文 TTS 音频(全平台共享缓存)。

## 关键能力
1. 应用端 3 层:类目卡片 → 文章列表 → 逐句详情;
2. 单句朗读 + 全文朗读(按句循环播放);
3. 用户系统语言决定"本地语言"展示(中文用户只看拼音 + 中文);
4. 阅读进度:记忆到第几句;
5. 管理端 3 层:类目 → 文章 → 编辑;
6. 5 语言 Tab 句子编辑 + 任意位置增删 + 重排;
7. 搜索:管理端类目层搜全局(标题 + 句子),文章层搜本类目,句子层搜本文;
8. 应用端搜索仅可见语言(中文 + 本地语);
9. 危险操作二次确认 + 未保存离开拦截。

## 业务规则
- 文章默认"待发布",发布即应用端可见;下架清空所有用户进度;
- 类目下有文章则禁止删除;
- 句子顺序删改自动重排,音频缓存键跟随;
- 多人编辑"后保存覆盖前保存"+ Toast 提示被覆盖;
- TTS:用户点播才生成 + 全平台共享 + 失败不限次重试。
