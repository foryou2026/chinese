## 0. 摘要

- 给谁：知语 app 端访客 / 学员（接触中国文化内容），与 admin 端的内容运营管理员。
- 做什么：以 12 个中国文化类目为骨架的双语对照文章浏览 + 句子级中文 TTS 朗读 + 阅读进度记忆；admin 端做类目统计 / 文章 CRUD / 5 语句子编辑 / 发布管理 / 全文搜索。
- 最大价值：把"用中文了解中国文化"做成"开箱即用"的轻量入口，降低用户首单试用门槛。
- 最大风险：多语言齐备 + 句子级 TTS 缓存 + 多管理员并发编辑这三件事一旦走偏，会出现"上线即脏"或"反复返工"。

## 1. 产品定义与目标

### 1.1 一句话定义

围绕 12 个固定类目的双语文章浏览 + 句子级中文朗读，让东南亚学员能"在没注册的前提下就先看见 / 听见中国文化"。

### 1.2 业务目标

| ID | 目标 | 度量口径 | 验证时点 |
|----|------|--------|---------|
| O1 | 访客首次访问可在 30 秒内完成"看一篇 + 听一句"正向体验 | 首次访问到首句 TTS 成功播放的时长 | 上线后 4 周 |
| O2 | 文章已发布闭环可走通 | 运营从"新建文章 → 5 语齐备 → 发布"的成功率 ≥ 95% | 上线后 4 周 |
| O3 | TTS 缓存命中率 | 句子级 TTS 缓存命中率 ≥ 95% | 持续 |
| O4 | 阅读进度跳转准确 | 用户回到上次阅读句子的命中率 = 100% | 持续 |

### 1.3 用户类型

| 类型 ID | 名称 | 画像（≤2 行） | 核心诉求 |
|--------|------|-----------|---------|
| U1 | 访客 / 学员（app） | 18-35 岁东南亚成年人；中 / 英 / 越 / 泰 / 印尼母语 | 想"用本国语言对照看中文，听标准普通话发音" |
| U2 | 内容运营（admin） | 内部团队；懂中文 | 想能稳定批量录入与发布 + 不被并发覆盖 |

## 2. 暗依赖警示

- "12 类目固定 + 不可删"是底座；任何下游若按"可变类目"建表 / 建路由都是错误。
- "句子粒度 TTS 缓存键"必须随句子序号变化而失效与重建；这件事在 §6 显式列出。
- 多管理员并发编辑使用"后写覆盖 + 提示"语义，不做强锁，下游不要默认乐观锁拒绝。

## 5. 业务场景（叙述）

> 仅做业务叙述。详细流程图、状态机交给 C03 信息架构阶段。

### 5.1 主要场景

- 访客首次正向体验：打开 app 首次点"发现中国"Tab，看到 12 张卡片，进入 01-03 任一类目，进入任意文章看到逐句对照中文 + 母语，点播放立即听到普通话朗读。
- 登录学员深度阅读：登录后回到上次的文章，自动跳到上次阅读的句子；点全文朗读从当前句开始按顺序播放。
- 运营内容生产：管理端选定类目 → 新建文章 → 5 语 Tab 录入 → 句子任意位置插入与重排 → 发布；学员端立即可见。

### 5.2 隐含的重点能力（供 C03 拆解）

- 句子级 TTS 缓存键生成与失效
- 多管理员并发编辑后写覆盖语义
- 软删 + 定期物理清理的可见性窗口
- 访客阅读进度的本地兜底与登录后合并
