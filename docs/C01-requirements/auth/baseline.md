## 0. 摘要

- 给谁：知语应用端（18-35 岁东南亚成年用户）与运营管理端（内部管理员）的全部账号 / 登录 / 会话 / 密码相关能力。
- 做什么：邮箱密码与 Google 一键的注册登录、会话治理（多设备上限）、密码自助找回 / 修改、登录守卫、个人资料维护。
- 最大价值：用最低门槛、最强守卫，把"能进得来 + 进得安全 + 跨端一致"的账户体验做成可信底座。
- 最大风险：双 surface 共用一套账号系统时角色 / 会话隔离与"管理员入口被普通用户误用"风险，需要在路由守卫 + 会话 + 审计层共同兜住。

---

## 1. 产品定义与目标

### 1.1 一句话定义

知语的统一账户与会话能力：让用户在 app 端自助注册登入、让管理员在 admin 端凭邀请账号安全登入，并由会话与守卫保证两端互不串扰。

### 1.2 业务目标

| ID | 目标 | 度量口径 | 验证时点 |
|----|------|--------|---------|
| O1 | 新用户首次完成注册 + 邮箱验证 + 登入 | 注册→首次登入完成率 ≥ 70% | 首版上线后 4 周 |
| O2 | 管理员登录失败率可控 | 单日"非管理员尝试登录管理端" / "全部管理端登录" ≤ 1% | 首版上线后 4 周 |
| O3 | 会话稳定性 | 30 天内被动登出（会话异常 / 错踢）≤ 0.5% | 持续 |
| O4 | 密码自助找回闭环可走通 | 用户进入"找回密码"后 24 小时内成功改新密码并登入 ≥ 80% | 首版上线后 4 周 |

### 1.3 用户类型

| 类型 ID | 名称 | 画像（≤2 行） | 核心诉求 |
|--------|------|-----------|---------|
| U1 | 普通学习者（app） | 18-35 岁东南亚成年人，移动优先，部分用 Google 帐户 | 想用最快的方式注册 / 找回密码并安全使用账号 |
| U2 | 运营管理员（admin） | 内部团队成员，邀请制账号，桌面优先 | 想以稳定凭据登录、被严格守卫，但不被会话频繁打断 |

## 2. 暗依赖警示

- 多 surface 复用同一账号时，"普通用户在管理端登录入口直接登入"必须被识别并礼貌阻断；这要求登录态成功后立即做角色校验并执行登出动作。已在 R-auth-003 / R-auth-010 中显式表达。
- 会话设备上限（3 台）需要按 surface 独立计数（app 端 3 台 + admin 端 3 台），避免某一端把另一端踢掉；已在 R-auth-005 显式声明。
- 不出现 P0 / P1 / 本期 / 二期 / MoSCoW 类分期表达；所有列入 §4 的 R-ID 平等对待。

## 3. 角色与权限（粗）

| 角色 ID | 名称 | 能做（一句话） | 不能做 |
|--------|------|-------------|--------|
| ROLE-USER | 普通用户 | 注册 / 登录 / 找回密码 / 修改密码 / 维护个人资料 / 进入受守卫的 app 页面 | 进入 admin 受守卫页面 |
| ROLE-ADMIN | 管理员 | 登录 / 找回密码 / 修改密码 / 退出 / 访问全部 admin 受守卫页面 | 在管理端自助注册；从管理端编辑普通用户密码 |

## 4. 需求清单（核心交付物）

> 所有 R-ID 平等交付。本表不设接口列与优先级列；技术映射统一交由 D 阶段做。

| R-ID | scope | 一句话 | 涉及角色 | 关联页面 | 验收要点 |
|------|:----:|--------|---------|----------|---------|
| R-auth-001 | app | 邮箱 + 密码注册账号，必须邮箱验证后才能首次登入 | ROLE-USER | `P-app-auth-002` / `003` / `004` | 注册成功后立即给出"待验证邮件"反馈；未验证不得登入 |
| R-auth-002 | app | Google 一键注册即登录，无需邮箱验证 | ROLE-USER | `P-app-auth-002` / `004` | 首次回调成功即建账户并登入；失败可重试 |
| R-auth-003 | both | 邮箱 + 密码登录；登录前置节流 + 禁用检查（admin 端额外校验管理员身份，非管理员立即登出并提示"请使用用户入口登录"） | ROLE-USER / ROLE-ADMIN | app: `P-app-auth-001` · admin: `P-admin-auth-001` | 普通用户在 admin 入口登入被阻断；提示文案 i18n 齐备 |
| R-auth-004 | both | 登录态使用 HttpOnly Cookie 维持；30 天滚动会话；写操作必须带 CSRF 令牌 | ROLE-USER / ROLE-ADMIN | 跨页（不可见） | 关闭浏览器不丢登录；跨域写操作无令牌即被拒 |
| R-auth-005 | both | 多设备活跃会话每 surface 各 3 台上限，第 4 台登入踢最早 | ROLE-USER / ROLE-ADMIN | app: `P-app-auth-001` 被踢提示 · admin: `P-admin-auth-001` 被踢提示 | 第 4 设备登入 10 秒内最早设备被强制退出 |
| R-auth-006 | both | 短时间内连续输错密码触发账号自锁；账号被禁用时全设备登出且无法登入 | ROLE-USER / ROLE-ADMIN | app: `P-app-auth-001` 错误态 · admin: `P-admin-auth-001` 错误态 | 锁定文案明确剩余时间；解禁 / 解锁后可恢复登入 |
| R-auth-007 | both | 忘记密码 → 邮件链接 → 设新密码 → 自动登入并踢其它设备；链接 15 分钟一次性 | ROLE-USER / ROLE-ADMIN | app: `P-app-auth-005` / `006` · admin: `P-admin-auth-002` / `003` | 链接过期 / 二次使用必须给出明确错误页与重发入口 |
| R-auth-008 | both | 已登录用户可修改密码（先验旧密）；可选"全部设备退出" | ROLE-USER / ROLE-ADMIN | app: `P-app-auth-008` · admin: `P-admin-auth-004` | 改密成功即生效；勾选"全部设备退出"后其它设备 30 秒内失效 |
| R-auth-009 | both | 用户可随时"本设备退出"或"全部设备退出" | ROLE-USER / ROLE-ADMIN | app: `P-app-auth-008` + 顶栏头像 · admin: `P-admin-auth-004` | 退出后回到登录页；选"全部退出"则所有设备同时失效 |
| R-auth-010 | both | 路由守卫：未登录访问受守卫页跳回该 surface 登录页并保留 redirect；admin 端额外校验管理员身份 | ROLE-USER / ROLE-ADMIN | 跨页（守卫层） | 登入后自动按 redirect 回到原页；非管理员被守卫挡住时文案与跳转都一致 |
| R-auth-011 | app | 已登录用户可查看 / 修改 profile：头像 / 显示名 / 偏好语言 | ROLE-USER | `P-app-auth-007` / `009` | 改完即生效；偏好语言变更影响应用端默认 UI 语言 |
| R-auth-012 | app | Google 回调失败 / 用户取消 → 静默回登录页；身份异常 → 提示并登出 | ROLE-USER | `P-app-auth-004` | 取消场景不报错，仅回登录页；身份异常给一次性 Toast |
| R-auth-013 | app | 邮箱验证 / 重置链接过期 → 给出"链接已过期"页与重新发起入口 | ROLE-USER | `P-app-auth-003` / `006` 失效态 | 过期页支持一键重发；不暴露邮箱是否存在 |
| R-auth-014 | app | 邮箱已注册时注册流程提示"已注册，请去登录或找回密码"；找回密码不暴露邮箱是否存在 | ROLE-USER | `P-app-auth-002` / `005` | 不可枚举账号是否存在；找回密码统一回执文案 |
| R-auth-015 | app | "重新发送验证邮件"按钮带 60 秒倒计时与节流 | ROLE-USER | `P-app-auth-003` | 倒计时未结束按钮置灰；超频请求被拒并给出剩余时间 |
| R-auth-016 | admin | 管理员账号通过运维 seed 流程创建，本条仅为契约存档（无产品功能） | ROLE-ADMIN | — | 文档化"管理员仅可由运维创建"，并明确不开放自助注册 |

## 5. 业务场景（叙述）

> 以下仅做业务叙述。详细业务流程、状态机、异常分支留给 C03 信息架构阶段。

### 5.1 主要场景

- 普通用户首次使用：在 app 登录页选择"邮箱注册"或"Google 一键登录"。邮箱注册需收信验证后才能进入；Google 注册回调成功即登入。登入成功进入下游业务。
- 管理员日常登入：在 admin 登录页凭邀请账号登入，系统校验"管理员"身份后进入工作台。普通用户若误用 admin 入口登入会被立即阻断并提示去用户入口。
- 自助找回密码：用户在登录页点击"忘记密码"，输入邮箱后收到一次性邮件链接（15 分钟内一次性有效），设置新密码后自动登入并踢出其它设备。
- 修改 / 维护账号：登录后可在个人中心修改密码 / 维护个人资料（仅 app 端含资料维护），可选择"本设备退出"或"全部设备退出"。

### 5.2 隐含的重点能力（供 C03 拆解）

- 跨 surface 共用账号但独立会话计数
- 节流 + 自锁 + 禁用三层防滥用
- 守卫层在 app / admin 各自的"登录页 redirect"约定
- 错误码 i18n 与文案一致性（5 语）

| 规则 ID | 描述 | 来源 R-ID | 边界值 |
|--------|------|---------|--------|

| BR-2 | Google 一键注册不再走邮件验证 | R-auth-002 | — |

## 8. 非功能性需求（NFR）

| 类型 | 指标 | 验收口径 |
|------|------|---------|
| 性能 | 登录页 TTFB | < 200ms（静态壳走 CDN） |
| 性能 | 登录请求时延 | p95 < 800ms |
| 可用性 | 任意 auth 接口 | ≥ 99.5% 月度可用率 |
| 国际化 | 错误码 / 文案 | zh / en / vi / th / id 全量 |
| 兼容 | 浏览器 | app 端覆盖主流移动浏览器；admin 端覆盖主流桌面浏览器 |
| 隐私 | 凭据存储 | 严禁在浏览器本地存储中保留长效令牌；严禁明文密码进入日志或审计 |
| 隐私 | 错误回执 | 不得通过响应差异让外部判断邮箱是否存在 |
| 审计 | 管理端关键动作 | 登录 / 改密 / 退出 100% 进入审计记录 |
| 无障碍 | 表单 | WCAG AA；可纯键盘完成；focus ring 可见 |
