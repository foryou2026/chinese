# 权限注册表

> 增量更新，标注 [本轮新增] / [本轮变更]。

## P1. 角色枚举

| 角色 ID | DB/JWT 字面量 | 显示名 | 职责 | 不能做 | 首次引入功能 | 增量标记 |
|---------|--------------|--------|------|--------|------------|---------|
| ROLE-ADMIN | `app_metadata.role = 'admin'` | 管理员 | 登录后台、管理业务数据 | 不能自行注册，需超管授予 | auth/account-entry | [本轮新增] |

### P1.1 多角色策略
- admin 系统仅有 ROLE-ADMIN 一种角色
- 一用户不能同时拥有多角色（admin 系统内）
- 后续若需分级（超管/普通管理员），在 C01 新增功能时扩展
