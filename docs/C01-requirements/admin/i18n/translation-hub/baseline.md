# 需求与权限基线

> **阶段**：C01
> **系统**：admin
> **冻结状态**：未冻结
> **本轮聚焦功能**：国际化翻译中心（翻译模块 + 配音模块，完全解耦）
> **权限文件**：角色见 docs/C01-requirements/admin/_shared/roles.md；矩阵见同目录 matrix.md

## 0. 摘要
- 给管理员 / 提供统一的多语言翻译与配音管理平台 / 最大价值：翻译与配音解耦，一站式控制 app 端所有多语言内容及语音资源 / 最大风险：AI 翻译质量不可控

## 1. 产品定义与目标
### 1.1 一句话定义
为管理员提供一站式国际化管理中心：翻译模块（翻译总览、文案翻译、内容翻译、翻译任务、翻译接口）、配音模块（配音总览、配音管理、音色管理、配音任务、配音接口）、全局模块（语言管理）三大分组完全解耦，用户可按需使用任一模块。

### 1.2 业务目标
| ID | 目标 | 度量口径 | 验证时点 |
|----|------|---------|---------|
| G-i18n-001 | 管理员可在后台管理多语言 | 成功启用/停用语言 | 上线即验证 |
| G-i18n-002 | 文案翻译覆盖率 ≥ 95% | 已翻译 key 数 / 总 key 数 | 每次翻译任务完成后 |

## 3. 业务场景（叙述）
### 3.1 翻译场景
管理员登录后台，进入「国际化管理」。侧边栏分为"翻译"、"配音"、"全局"三组菜单。首先看到翻译总览，展示各语言翻译覆盖率。

点击「全局 → 语言管理」，看到约 50 种主流语言（涵盖东亚、东南亚、南亚、西欧、北欧、东欧、中东/非洲、美洲变体），中文已启用且不可停用，英文已启用且不可停用（枢纽语言）。管理员启用越南语。

进入「翻译 → 翻译接口」，看到供应商列表。点击「新增供应商」，填写供应商名称（如"Claude"）、API 端点、密钥。进入供应商详情后添加模型：名称"Claude Sonnet"、模型 ID `claude-sonnet-4-20250514`、请求参数 JSON。配置翻译提示词模板。测试翻译效果。

进入「翻译 → 文案翻译」，系统已自动同步代码中所有 i18n key 及中文原文。管理员切换右侧语言为英文，选择翻译模型"Claude Sonnet"，点击「翻译全部待翻译」，系统创建翻译任务。翻译完成后状态变为"已翻译"。管理员可直接编辑已翻译内容，编辑后状态保持"已翻译"。最后点击「发布」生效。

进入「翻译 → 内容翻译」，看到所有业务表列表。点击进入课程表，看到字段列表。管理员将 `title` 和 `description` 设为"需要翻译"，为 `title` 指定翻译模型"Claude Sonnet"。进入 `title` 字段翻译详情后批量翻译。系统按翻译链规则执行：先查英文翻译→无则先翻英文→再翻目标语言。界面显示中文原文和目标语言对照。

### 3.2 配音场景（可选，完全独立于翻译）
管理员如果需要配音功能，进入「配音 → 配音总览」。看到配音模块全局开关，开启后展示各语言配音覆盖率。

进入「配音 → 配音接口」，看到 TTS 供应商列表。新增供应商"Azure TTS"，填写 API 端点和密钥。进入详情添加音色模型：名称"Xiaoxiao"、voice_id `zh-CN-XiaoxiaoNeural`、语言中文、参数 JSON。

进入「配音 → 音色管理」，查看所有音色配置。按语言分组展示，可试听测试。

进入「配音 → 配音管理」，看到数据表列表（同内容翻译结构）。点击课程表进入字段列表页。为 `title` 字段开启配音并指定音色模型。点击进入 `title` 字段配音详情页，查看每条记录的配音文件，可播放、重新生成、上传替换。选择音色后批量生成配音。

进入「配音 → 配音任务」查看配音批次的执行进度和历史。

### 3.3 解耦说明
翻译模块、配音模块、全局模块三者完全独立。语言管理是三者的共用部分，置于全局分组。不需要配音时只需忽略配音菜单组即可。

### 3.4 隐含重点能力
- 代码 i18n 资源文件与数据库的双向同步机制
- AI 翻译的异步任务队列与进度反馈
- 翻译状态的自动流转（源文变更→自动回退为待翻译）
- 字段级与表级配置的优先级合并逻辑
- 配音模块的独立生命周期（与翻译完全解耦）
- 接口配置的通用化（多供应商多模型，非硬编码平台）
- 枢纽语言翻译链（所有翻译以英文为媒介）
- 层级导航：配音管理同内容翻译结构（表→字段→详情）
