# 需求与权限基线

> **阶段**：C01
> **系统**：admin
> **冻结状态**：未冻结
> **本轮聚焦功能**：国际化翻译中心（语言管理/UI文案翻译/数据库内容翻译/配音管理）
> **权限文件**：角色见 docs/C01-requirements/admin/_shared/roles.md；矩阵见同目录 matrix.md

## 0. 摘要
- 给管理员 / 提供统一的多语言翻译管理平台 / 最大价值：一站式控制 app 端的所有多语言内容 / 最大风险：AI 翻译质量不可控

## 1. 产品定义与目标
### 1.1 一句话定义
为管理员提供一站式国际化管理中心：管理支持语言、控制 UI 文案翻译、管理数据库内容翻译、可选配音管理，所有翻译由 AI 驱动。

### 1.2 业务目标
| ID | 目标 | 度量口径 | 验证时点 |
|----|------|---------|---------|
| G-i18n-001 | 管理员可在后台管理多语言 | 成功启用/停用语言 | 上线即验证 |
| G-i18n-002 | UI 文案翻译覆盖率 ≥ 95% | 已翻译 key 数 / 总 key 数 | 每次翻译任务完成后 |
| G-i18n-003 | 数据库内容翻译可按表/字段粒度控制 | 字段级配置功能可用 | 上线即验证 |
| G-i18n-004 | AI 翻译一键触发 | 批量翻译任务执行成功率 ≥ 90% | 上线首周 |

### 1.3 用户类型
| 类型 ID | 名称 | 画像 | 核心诉求 |
|---------|------|------|---------|
| UT-admin-001 | 运营管理员 | 内部员工，管理多语言内容 | 快速查看翻译状态，一键触发翻译，管理配音 |

## 3. 业务场景（叙述）
### 3.1 主要场景
管理员登录后台，进入「国际化管理」模块。首先看到翻译总览仪表盘，展示各语言翻译覆盖率。点击「语言管理」，看到全球约 50 种主流语言列表，默认中文已启用且不可停用。管理员启用英文和越南语。

进入「UI 文案翻译」，系统已自动同步代码中所有 i18n key 及中文原文。管理员切换右侧语言为英文，看到大量"待翻译"标签。点击「全部翻译」，系统创建翻译任务，AI 自动将中文翻译为英文。翻译完成后，管理员抽查几条，手动修正不准确的翻译，点击「发布」生效。

进入「数据库内容翻译」，看到所有业务表列表。点击进入课程表，看到字段列表。管理员将 `title` 和 `description` 字段设为"需要翻译"，源语言为中文；将 `course_code` 设为"不需要翻译"。进入 `title` 字段翻译详情，看到所有课程标题的中文原文和英文翻译对照。点击「翻译全部待翻译」，AI 批量翻译。

（可选）进入配音管理，为越南语配置 TTS 接口，对已翻译的课程标题触发配音生成。

### 3.2 隐含重点能力
- 代码 i18n 资源文件与数据库的双向同步机制
- AI 翻译的异步任务队列与进度反馈
- 翻译状态的自动流转（源文变更→自动标记过期）
- 字段级与表级配置的优先级合并逻辑
- 配音模块的独立生命周期
