# 需求与权限基线

> **阶段**：C01
> **系统**：admin
> **冻结状态**：未冻结
> **本轮聚焦功能**：国际化翻译中心（翻译模块 + 配音模块，完全解耦）
> **权限文件**：角色见 docs/C01-requirements/admin/_shared/roles.md；矩阵见同目录 matrix.md

## 0. 摘要
- 给管理员 / 提供统一的多语言翻译与配音管理平台 / 最大价值：翻译与配音解耦，一站式控制 app 端所有多语言内容及语音资源 / 最大风险：AI 翻译质量不可控

## 1. 产品定义与目标
### 1.1 一句话定义
为管理员提供一站式国际化管理中心：翻译模块（语言管理、文案翻译、内容翻译、翻译任务、翻译接口）和配音模块（配音管理、配音任务、配音接口）完全解耦，用户可按需使用任一模块。

### 1.2 业务目标
| ID | 目标 | 度量口径 | 验证时点 |
|----|------|---------|---------|
| G-i18n-001 | 管理员可在后台管理多语言 | 成功启用/停用语言 | 上线即验证 |
| G-i18n-002 | 文案翻译覆盖率 ≥ 95% | 已翻译 key 数 / 总 key 数 | 每次翻译任务完成后 |
| G-i18n-003 | 内容翻译可按表/字段粒度控制 | 字段级配置功能可用 | 上线即验证 |

## 3. 业务场景（叙述）
### 3.1 翻译场景
管理员登录后台，进入「国际化管理」。侧边栏分为"翻译"和"配音"两组菜单。首先看到翻译总览，展示各语言翻译覆盖率。点击「语言管理」，看到约 50 种主流语言（涵盖东亚、东南亚、南亚、西欧、北欧、东欧、中东/非洲、美洲变体），默认中文已启用且不可停用。管理员启用英文和越南语。

进入「文案翻译」，系统已自动同步代码中所有 i18n key 及中文原文，按业务域分组（common、auth、nav、course 等，域从 key 的一级前缀自动提取）。管理员切换右侧语言为英文，点击「翻译全部待翻译」，系统创建翻译任务。翻译完成后点击「发布」生效。

进入「内容翻译」，看到所有业务表列表。点击进入课程表，看到字段列表（纯翻译配置，不含配音列）。管理员将 `title` 和 `description` 设为"需要翻译"，进入翻译详情后批量翻译。

进入「翻译接口」，配置自定义翻译供应商：填写供应商名称、API 端点、密钥、模型 ID，在代码编辑器中配置请求参数 JSON，编写翻译提示词模板（支持 {source_lang}、{target_lang}、{text}、{context} 变量），测试翻译效果。

### 3.2 配音场景（可选，完全独立于翻译）
管理员如果需要配音功能，进入「配音管理」。首先开启配音模块全局开关。在"配音配置"Tab 中配置哪些表/字段需要配音（类似内容翻译的表→字段层级）。在"音色管理"Tab 中按语言配置默认音色参数。在"配音文件"Tab 中查看管理已生成的配音文件（支持按字段筛选和中文搜索）。

进入「配音接口」，配置自定义 TTS 供应商：填写供应商名称、API 端点、密钥，在代码编辑器中配置请求参数 JSON，按语言配置音色标识和参数，测试生成效果。

进入「配音任务」查看配音批次的执行进度和历史。

### 3.3 解耦说明
如果用户不需要配音功能，只需不点击配音模块的三个菜单即可。翻译模块独立运行，不受配音模块影响。语言管理是两个模块的共用部分。

### 3.4 隐含重点能力
- 代码 i18n 资源文件与数据库的双向同步机制
- AI 翻译的异步任务队列与进度反馈
- 翻译状态的自动流转（源文变更→自动标记过期）
- 字段级与表级配置的优先级合并逻辑
- 配音模块的独立生命周期（与翻译完全解耦）
- 接口配置的通用化（自定义供应商，非硬编码平台）

## 10. 增量融合报告
