## 0. 摘要
- 给管理员 / 提供安全可控的后台登录与密码管理 / 最大价值：保障运营系统安全访问 / 最大风险：暴力破解与未授权访问

## 1. 产品定义与目标
### 1.1 一句话定义
为内部运营人员提供邮箱+密码的后台安全登录入口，含找回密码与登录态密码修改能力。

### 1.2 业务目标
| ID | 目标 | 度量口径 | 验证时点 |
|----|------|---------|---------|
| G-admin-auth-001 | 管理员可通过邮箱密码登录后台 | 登录成功率 ≥ 99% | 上线首周 |

### 1.3 用户类型
| 类型 ID | 名称 | 画像 | 核心诉求 |
|---------|------|------|---------|
| UT-admin-001 | 运营管理员 | 内部员工，PC 办公，日常使用管理后台 | 快速安全登录、密码遗忘时可自助恢复 |
| UT-admin-002 | 超级管理员 | 技术负责人，管理其他管理员账号 | 账号安全可控、操作可审计 |

## 3. 业务场景（叙述）
### 3.1 主要场景
管理员打开后台域名，进入登录页。输入已注册的邮箱和密码，点击登录。系统验证凭证并校验 JWT 中 role=admin，验证通过后跳转 Dashboard。若密码遗忘，管理员点击"忘记密码"链接，输入邮箱后系统发送 6 位验证码至邮箱，管理员输入验证码校验通过后进入重置密码页面设置新密码。密码修改/重置成功后，该账号已登录的其他设备自动下线。每个账号最多同时登录 2 台设备，超出时自动下线最早登录的设备。登录状态下，管理员可通过右上角头像下拉菜单进入修改密码或退出登录。

### 3.2 隐含重点能力（供 C02 拆解）
- 前端表单实时校验（邮箱格式、密码强度、确认密码一致性）
- 登录按钮防重复提交（loading 态禁用）
- 验证码发送倒计时（60 秒冷却防频繁触发）
- Token 过期时的全局拦截与静默刷新
- 页面级路由守卫（未登录重定向登录页）

## 4. 业务规则
| 规则 ID | 描述 | 来源 R-ID | 边界值 |
|---------|------|----------|--------|
| BR-001 | 密码长度 ≥ 8，含字母+数字 | R-auth-009 | 最短 8 字符 |
| BR-002 | 登录连续失败 5 次锁定 30 分钟；前 2 次仅提示错误，第 3 次起显示失败计数 | R-auth-002 | 阈值 3 次开始显示计数，5 次锁定/30 分钟 |
| BR-003 | 验证码 5 分钟有效 | R-auth-003 | 5 分钟 |
| BR-004 | 验证码 60 秒内不可重复发送 | R-auth-003 | 60 秒冷却 |
| BR-009 | 密码修改/重置后其他设备自动下线 | R-auth-004, R-auth-005 | 即时 |
| BR-010 | 每账号最多同时登录 2 台设备 | R-auth-011 | 超出时下线最早设备 |
| BR-005 | Access Token 有效期 1 小时 | R-auth-007 | 3600 秒 |
| BR-006 | Refresh Token 有效期 7 天 | R-auth-007 | 7 天 |
| BR-007 | 修改密码需验证旧密码 | R-auth-005 | 无 |
| BR-008 | 新密码不能与旧密码相同 | R-auth-005 | 无 |
