# 04 · 文案口吻 voice & tone

> **阶段**：B03-X  
> **下游**：所有 C03 N 4 态文案、所有 i18n key 文案、邮件模板、错误码 i18n

---

## 0. 摘要

- 基本口吻 = **平等、直接、温和**；不卖萌、不耍宝、不羞辱用户。
- **不滥用感叹号**：通篇仅"成功 / 鼓励"语境允许；错误 / 警告 / 提示一律用句号或问号。
- 5 语全量本地化；vi / th / id 不直译，由 native 校对。

---

## 1. 基本口吻（Voice 不变 · Tone 随场景变）

> **Voice 是恒定的人格**：克制、专业、温暖、稍微聪明（懂用户在学什么）。
> **Tone 是随场景调整的情绪**：成功时朴素鼓励 / 错误时温和指明 / 等待时简短陪伴 / 危险时严肃确认。

### 1.1 Voice 三句话

1. 我们说话**像懂教育的朋友**，不像后台机器人。
2. 我们说话**懂上下文**：你在错的同一个字第 3 次，我们不会说"哎呀又错啦"，而是说"这个字是高频混淆点，多数人需要 5 次以上才能稳定"。
3. 我们**不假装是真人**：不发"在吗？""你今天还没学习哦~"这种类人话术；通知就是通知。

### 1.2 Tone 表

| 场景 | Tone | 文案示例（zh）| 反例 |
|------|------|--------------|------|
| 成功（保存 / 提交）| 朴素 | "已保存" / "已发送验证邮件" | ❌ "保存成功啦！🎉 您真棒！" |
| 鼓励（连胜 / 完成单元）| 适度肯定 | "今天第 5 个新字，节奏很稳" | ❌ "天才！你太厉害了！" |
| 错误（表单 / 网络）| 温和明确 | "邮箱或密码错误" / "网络连接失败，请重试" | ❌ "出错了！请联系管理员！！" |
| 警告（高危操作前）| 严肃直接 | "此操作不可撤销，确认删除「{name}」吗？" | ❌ "⚠️⚠️⚠️ 注意注意！" |
| 等待 | 简短陪伴 | "正在生成…" / "正在保存…" | ❌ "请耐心等待"（多余）|
| 空状态 | 引导而非道歉 | "还没有课程，点右上「新建」开始" | ❌ "抱歉，暂无数据" |
| 学习反馈 - 对 | 朴素 | "对了" / "做得不错" | ❌ "Perfect！💯" |
| 学习反馈 - 错 | 不羞辱 | "再试一次。这个字的拼音是 `xué`" | ❌ "错了！你怎么连这个都不会！" |
| 学习反馈 - 高频错 | 共情 | "这个字是高频混淆点。提示：右边是 `子`，不是 `了`" | ❌ "又错了？这个字这么简单" |

## 2. 禁用词 / 慎用词

| 禁用 | 替代 |
|------|------|
| "哎呀" / "哦哦" / "啦" / "嘛" 等口语虚词 | 删 |
| "亲" / "宝宝" / "小可爱" 等称呼 | 不用任何称呼 |
| "请耐心等待" | "正在 ..." |
| "操作成功" | 说清做了什么："已保存" / "已发送" / "已删除" |
| "网络异常" | "网络连接失败，请重试" |
| "系统错误" / "Oops" | "服务暂时不可用，请稍后重试。错误码 {code}" |
| "您" 在年轻人语境里 | 一律用"你"；管理端可用"您" |
| 大量感叹号 / 颜文字 / emoji | 删；学习内容中允许出现拼音四声标记 |

## 3. 错误提示三件套（每条错误必备）

1. **是什么**：清晰说出错的事。
2. **为什么**（如果用户能改的话）：说出可见原因。
3. **怎么办**：给出下一步。

**例**：

| 错误码 | 仅"是什么" ❌ | 三件套 ✅ |
|--------|--------------|----------|
| `AUTH_INVALID_CREDENTIALS` | "登录失败" | "邮箱或密码错误。请检查后重试，或点击「忘记密码」重置。" |
| `AUTH_LOGIN_RATE_LIMITED` | "请稍后" | "失败次数过多，请 {minutes} 分钟后重试。如不是你本人操作，请尽快重置密码。" |
| `AUTH_ACCOUNT_DISABLED` | "账号异常" | "账号已被停用，原因：{reason}。如有疑问，请联系客服。" |

## 4. 5 语本地化原则

1. **不直译**：vi / th / id 必须本土化；
2. **长度差 + 30%**：UI 必须为最长语言（vi / id）预留空间；
3. **示例汉语 / 拼音**：作为学习内容，不进 i18n，直接走业务 jsonb；
4. **错误码文案**：5 语全量在 `packages/shared-i18n/<locale>/auth.json`；缺一个 CI 报错；
5. **AI 翻译占位**：所有 vi / th / id 文案上线前由 native 校对；自动翻译的条目标 `needs_review: true`；
6. **数字 / 日期 / 货币**：用 `Intl.NumberFormat` / `Intl.DateTimeFormat`，禁止手写格式串。

---

## 5. 邮件 / 推送（适用 admin 端用户管理 + auth 邮件 + 通知）

- 主题：≤ 30 字，直说事；
- 正文开头**不寒暄**：直接说"我们收到你的请求..."；
- 落款：固定"知语 团队 / Zhiyu Team / Đội Zhiyu / ทีม Zhiyu / Tim Zhiyu"（5 语）；
- 不放 emoji 不放 GIF，不放营销链接（除"取消订阅"）；
- 不在邮件正文重复显示明文密码 / token。
