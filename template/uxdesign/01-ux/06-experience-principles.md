# 体验决策原则（6 条）

> 每条都必须可用来"否决一个具体设计决定"。
> 当原则冲突时按编号排序，P1 > P2 > … > P6。

---

## P1. 温度但有边界

- **表述**：界面必须有"温度"（暖墨色 + 玻璃釉光 + 行楷点睛），但温度只服务于阅读舒适与品牌识别，**不允许溢出为装饰**。
- **判定**：当"加点中国风元素让它更好看"与"信息表达清晰"冲突时，选清晰；当"加点圆角更可爱"与"克制锋利"冲突时，选克制。
- **可否决**：在表格里加水墨纹底 → 否决（伤害可读性）；按钮全做胶囊形 → 否决（违反锋利）。

## P2. 信息密度优先于呼吸感

- **表述**：默认密度按"专业操作者一屏完成尽可能多事"配置；呼吸感通过"玻璃 + 釉光"营造，不通过"加大留白"营造。
- **判定**：当"留白让画面更高级"与"一屏少 30% 信息"冲突时，选信息。
- **可否决**：把表格行高从 44 改 64「为了高级感」 → 否决；H1 上下加 96px 留白 → 否决。

## P3. 玻璃为唯一表面语言

- **表述**：所有"卡片 / 面板 / 弹层 / TopBar / Tag"都使用毛玻璃 `backdrop-filter`；不允许出现"实色卡 + 1px 灰边"或"重投影立体卡"两种异类表面。
- **判定**：例外仅有 3 处：① 输入框内底（保可读）；② 暗 hero 横幅内文字底（保对比）；③ 长列表行 > 50 时降级实色（保性能）。
- **可否决**：引入 AntD 默认 Card → 否决（必须经过玻璃 token 二次封装）；自定义实色卡片 → 否决。

## P4. 文案直说，不卖萌

- **表述**：所有文案禁用感叹号、emoji、煽情词；错误必须给"下一步"，空态必须给"路径"。
- **判定**：见 `04-voice-tone.md`。
- **可否决**：文案 PR 出现「太棒啦」「Oops」「💪」 → 否决。

## P5. 动效服务反馈，不服务表演

- **表述**：动效仅用于"状态切换的过渡平滑"与"焦点引导"，不用于"装饰 / 营销"；时长 ≤ 250ms，缓动统一。
- **判定**：bounce / 视差 / 入场动画 / 字符逐个出现 → 一律否决。
- **可否决**：登录页 H1 字符依次淡入 → 否决（违反沉静）。

## P6. 流体仅为气韵，不为装饰

- **表述**：Three.js 流体波浪只允许在登录 / 欢迎 / Hero / 空状态 / 错误页出现，且每屏 ≤ 1 个；几何体 / 粒子 / 物理模拟一律禁止。
- **判定**：见 `design/02-design-system/` 中 Three.js 章节（继承自 prstory `spec/05`）。
- **可否决**：业务详情页右下角加旋转 3D logo → 否决；表格背景加流动粒子 → 否决。

---

## 原则优先级（冲突仲裁）

```
P1 温度但有边界   ──┐
                    │
P2 信息密度优先   ──┤
                    │── 当冲突时按此顺序仲裁，编号小者胜
P3 玻璃唯一表面   ──┤
                    │
P4 文案直说       ──┤
                    │
P5 动效克制       ──┤
                    │
P6 流体仅气韵     ──┘
```

---

## 应用示例

| 场景 | 适用原则 | 决定 |
|------|---------|------|
| 设计师想给"成功"提示加 🎉 | P4 | 否决，改文字「已保存」 |
| 工程师想用 AntD Modal 默认皮 | P3 | 否决，必须用 `.modal-card` 玻璃 |
| 产品想首页加视差滚动展示模块 | P5 | 否决，改静态分屏 |
| 设计师在表格里加宣纸纹理底 | P1 vs P2 | P2 胜，否决 |
| 主理人想换主题色到朱砂红 | P3 不冲突 | 通过 `data-accent="cinnabar"` 切换，按钮自动跟随 |
| 设计师在业务详情页加 Three.js | P6 | 否决，仅门面允许 |
