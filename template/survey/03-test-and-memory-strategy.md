### 1.2 各层级测试参数对比

| 维度 | 节测试 | 章测试 | 阶段测试 | 错题集考试 |
|------|-------|-------|---------|----------|
| **触发时机** | 每节学习完毕后 | 每章所有节完成后 | 每阶段所有章完成后 | 用户主动选择 |
| **题目数量** | 10题 | 25-35题 | 60-80题 | 10-50题(用户选) |
| **时间限制** | 5-8分钟 | 15-20分钟 | 45-60分钟 | 按题量推算 |
| **通过标准** | 70%正确率 | 70%正确率 | 70%正确率 | 无（练习性质） |
| **失败处理** | 重做本节/跳过 | 重做/跳过+标记薄弱章 | 重做/定向复习 | N/A |
| **题型覆盖** | 3-4种L1题型 | 6-8种L1题型 | 全部9种免费L1题型 | 原题型 |
| **考察范围** | 仅本节知识点 | 本章全部知识点 | 本阶段全部+前置回顾 | 仅错题 |
| **是否限制进度** | 否(可跳过) | 否(可跳过) | 是(必须通过才能解锁下一阶段) | 否 |

## 二、节测试(Section Test)详细设计

### 2.1 题量与构成

每节测试固定 **10道题**，从该节涉及的知识点中抽题：

| 题型(L1) | 题数 | 出题策略 |
|----------|------|---------|
| 选择题 | 3-4题 | 覆盖本节核心词汇/语法，优先使用高频L3变体 |
| 配对题 | 1-2题 | 汉字↔拼音 或 词语↔释义 |
| 填空题 | 2-3题 | 拼音填写 或 语境补词 |
| 判断题 | 1-2题 | 拼音正误 或 词义正误 |
| 排序题 | 0-1题 | 词语排句（Stage 2+才引入） |

### 2.2 各阶段节测试差异

| 阶段 | 题目来源 | 特殊规则 |
|------|---------|---------|
| Stage 1 | 100%本节新学内容 | 仅选择+配对+判断，最简单题型 |
| Stage 2 | 80%本节 + 20%前序节回顾 | 引入填空和排序 |
| Stage 3 | 70%本节 + 30%回顾 | 引入成语题、翻译题 |
| Stage 4 | 60%本节 + 40%回顾 | 回顾比例提升，强化长期记忆 |
| Stage 5-6 | 50%本节 + 50%回顾 | 半数题目来自历史内容，防止遗忘 |

### 2.3 出题逻辑（题库抽取规则）

```
节测试出题引擎:
1. 确定本节的知识点集合 K = {k1, k2, ..., kn}
2. 从K中按权重抽取核心知识点（新词汇 > 新语法 > 新汉字）
3. 为每个知识点匹配适合的L1题型 → L2知识类型 → L3变体
4. 如有回顾比例，从该章前序节或前序章的错题/低掌握度知识点中补充
5. 打散题目顺序，确保不连续出现同一题型
6. 生成选项干扰项（形近字/同音字/近义词/语法混淆项）
```

### 4.3 阶段测试出题范围

```
阶段测试知识点来源:
├── 本阶段全部知识点      ← 60% 权重
├── 前一阶段核心知识点    ← 25% 权重（验证长期记忆）
└── 更早阶段的高频错点    ← 15% 权重（消灭顽固弱点）
```

### 4.4 未通过处理机制

```
阶段测试未通过 (< 70%):
├── 生成《薄弱分析报告》
│   ├── 按知识类型统计正确率
│   ├── 标记最薄弱的3-5个知识点
│   └── 推荐针对性复习路径
│
├── 解锁定向复习模式
│   ├── 自动生成薄弱点的额外练习
│   └── 完成后可重新参加测试
│
└── 重考规则
    ├── 不限重考次数
    ├── 每次重考从题库重新抽题（非原题）
    └── 两次测试间隔至少2小时（防止刷记忆答案）
```

## 五、错题集(Wrong Answer Collection)系统

### 5.1 错题收录规则

| 来源 | 收录条件 | 收录信息 |
|------|---------|---------|
| 节测试 | 答错即收 | 原题+正确答案+用户答案+涉及知识点+错误时间 |
| 章测试 | 答错即收 | 同上 |
| 阶段测试 | 答错即收 | 同上 |
| 练习节 | 答错即收 | 同上 |

### 5.2 错题状态机

```
错题生命周期:
┌──────────┐    连续答对2次     ┌──────────┐    连续答对3次     ┌──────────┐
│  新错题   │ ────────────────→ │  复习中   │ ────────────────→ │  已掌握   │
│  (Red)   │                   │ (Yellow) │                   │ (Green)  │
└──────────┘                   └──────────┘                   └──────────┘
      ▲                              │                              │
      │          再次答错             │          再次答错             │
      └──────────────────────────────┘──────────────────────────────┘
```

- **新错题(Red)**：刚答错，需要尽快复习
- **复习中(Yellow)**：已在错题练习中答对2次，但还需巩固
- **已掌握(Green)**：连续答对3次（间隔至少1天），从活跃错题中移出
- **任何阶段再答错**：立即回退到"新错题"状态

### 5.3 错题练习模式

| 模式 | 说明 | 题量 |
|------|------|------|
| 快速练习 | 随机10-20题Red/Yellow错题 | 10/20题可选 |
| 知识类型专练 | 按知识类型筛选（如"语法错题"） | 10-30题 |
| 章节专练 | 按章节筛选错题 | 该章全部错题 |
| 高频错题 | 答错次数最多的Top20 | 20题 |
| 错题考试 | 正式计时测试，用错题组卷 | 用户选10/20/30/50题 |

## 六、记忆曲线(Spaced Repetition)复习策略

### 6.1 艾宾浩斯遗忘曲线核心原理

```
记忆保持率(%)
100│●
   │ ╲
 80│   ╲
   │     ╲
 60│       ╲
   │         ╲
 40│           ╲━━━━━━━━━━━━━━━━━━━━━ (无复习: 1天后仅剩33%)
   │
 20│
   │
  0┼────────────────────────────────────→ 时间
   0    20m   1h    9h   1d   2d   6d  31d

关键发现:
• 学习后20分钟，遗忘率42%
• 学习后1小时，遗忘率56%
• 学习后1天，遗忘率67%（仅保留33%！）
• 学习后6天，遗忘率75%
• 学习后1个月，遗忘率79%

每次复习都会"重置"遗忘曲线，且下一次遗忘速度变慢
```

### 6.2 我们的间隔复习时间表

综合 SM-2 算法、Leitner Box 和实际 App 场景，采用 **1-3-7-14-30** 核心节奏：

| 复习轮次 | 间隔 | 触发时机 | 预期保持率 | 复习形式 |
|---------|------|---------|----------|---------|
| R0 | 即时 | 学习节结束时 | ~100% | 节测试(10题) |
| R1 | 1天后 | 次日推送 | ~33%→恢复到~85% | 迷你复习(5题) |
| R2 | 3天后 | 第3天推送 | ~60%→恢复到~90% | 迷你复习(5题) |
| R3 | 7天后 | 第7天推送 | ~70%→恢复到~92% | 迷你复习(5题) |
| R4 | 14天后 | 第14天推送 | ~80%→恢复到~95% | 迷你复习(5题) |
| R5 | 30天后 | 第30天推送 | ~85%→恢复到~97% | 章测试(如恰好完成章) |
| R6 | 60天后 | 第60天推送 | ~90%→近永久 | 融入阶段测试 |

### 6.3 迷你复习(Mini Review)设计

迷你复习是记忆曲线的核心执行单元，每次仅 **5题 / 2-3分钟**：

```
迷你复习触发逻辑:
1. 系统每天检查: 有哪些知识点到了复习时间点？
2. 按优先级排序: Red错题 > 新学(R1) > R2到期 > R3到期 > ...
3. 从到期知识点中抽取5个，生成5道题
4. 用户答题:
   ├── 答对 → 该知识点进入下一个间隔(如R1→R2)
   └── 答错 → 该知识点重置为R1(1天后再复习) + 加入错题集
5. 日推送策略:
   ├── 早上: 推送昨天R1到期的知识点
   ├── 傍晚: 推送R2/R3到期的知识点
   └── 每日上限: 最多3组迷你复习(15题)，避免复习疲劳
```

### 6.4 SM-2 自适应难度算法

在迷你复习中，根据用户反馈调整间隔倍数：

```
核心公式:
  next_interval = current_interval × ease_factor

ease_factor 调整规则:
  初始值: 2.5
  答对(Easy): ease_factor += 0.15  (最大不超过3.0)
  答对(Good): ease_factor 不变
  答对(Hard): ease_factor -= 0.20  (最小不低于1.3)
  答错:       ease_factor -= 0.30  (最小不低于1.3), 重置interval=1天

示例:
  知识点"把"字句，ease_factor=2.5
  R1=1天后复习，答对(Good) → R2 = 1 × 2.5 = 2.5天 ≈ 3天
  R2=3天后复习，答对(Easy) → R3 = 3 × 2.65 = 7.9天 ≈ 8天
  R3=8天后复习，答错 → 重置! R1=1天后，ease_factor降为2.2
```

### 6.5 复习与新课的平衡

| 场景 | 每日推荐分配 | 时间 |
|------|------------|------|
| 正常学习日 | 1节新课(8-15m) + 1-2组迷你复习(4-6m) | 12-21分钟 |
| 复习积压日 | 0节新课 + 3组迷你复习(9m) | 9分钟 |
| 测试日 | 章/阶段测试 | 15-60分钟 |
| 错题强化日 | 错题专练20题 | 10-15分钟 |

**核心原则**：复习优先于新课。如果当日到期复习量超过15题，暂停新课学习，优先完成复习。这是 Duolingo 和 Anki 的共同最佳实践。

## 七、测试题库规模估算

### 7.1 每节需要的题库深度

为了支持复习、重考和防止题目重复，每个知识点需要多道不同的题目：

| 用途 | 每知识点需要的题数 | 说明 |
|------|----------------|------|
| 节测试 | 2-3道 | 确保重做时不完全重复 |
| 迷你复习(R1-R6) | 6道 | 每次复习抽1道不同的 |
| 章测试 | 1-2道 | 从知识点池中抽取 |
| 阶段测试 | 1-2道 | 从知识点池中抽取 |
| 错题变体 | 2-3道 | 错题练习时用变体题 |
| **总计** | **~12-16道/知识点** | |

### 7.2 全系统题库总量估算

| 知识类型 | 知识点数 | 每知识点题数 | 题库总量 |
|---------|---------|------------|---------|
| 拼音(声韵母+声调) | ~320个 | ×10道 | ~3,200题 |
| 偏旁部首 | ~214个 | ×8道 | ~1,700题 |
| 汉字 | ~1,800个 | ×12道 | ~21,600题 |
| 词汇 | ~5,400个 | ×15道 | ~81,000题 |
| 成语 | ~300个 | ×10道 | ~3,000题 |
| 语法点 | ~430个 | ×12道 | ~5,160题 |
| 短句 | ~800个 | ×8道 | ~6,400题 |
| 对话 | ~50场景 | ×6道 | ~300题 |
| 阅读 | ~300篇 | ×5道 | ~1,500题 |
| **总计** | — | — | **~123,860题** |

> **好消息**：你的架构里有 79种L3变体，可以为同一个知识点自动生成多种题型变体。配合 AI 批量生成干扰项（形近字/近义词/语法混淆项），~12万题的题库是完全可行的。

### 7.3 题库生产策略

| 阶段 | 策略 | 预计题量 |
|------|------|---------|
| **MVP阶段** | Stage 1-2 手动精编 + AI辅助生成干扰项 | ~15,000题 |
| **扩展阶段** | Stage 3-4 用 AI 批量生成初稿 → 人工审核 | +40,000题 |
| **完善阶段** | Stage 5-6 全AI生成 + 人工抽检 | +68,000题 |

## 八、各层测试的题型L3变体使用策略

### 8.1 节测试使用的L3变体（简单、快速）

| 知识类型 | 推荐L3变体 | 原因 |
|---------|----------|------|
| 拼音 | 汉字选拼音, 听音选拼音, 拼音正误 | 最直观的认知检验 |
| 汉字 | 拼音选汉字, 释义选汉字, 汉字↔拼音配对 | 快速识别 |
| 词汇 | 词义选择, 语境选词, 词语↔释义配对 | 核心语义理解 |
| 语法 | 语法词填选, 选正确句 | 即时语法应用 |
| 成语 | 成语选义, 含义选成语 | 基础理解 |

### 8.2 章测试使用的L3变体（全面、综合）

在节测试基础上增加：
- 填空题：语境补词, 翻译填词
- 排序题：词语排句, 对话排序
- 听力题：听句选义, 听词选义
- 翻译题：英中选择, 中英选择
- 听写题：听写词语

### 8.3 阶段测试使用的L3变体（高难度、综合能力）

在章测试基础上增加：
- 复合听力：短对话选择, 长段落选择, 听对话答题
- 高级填空：听音写拼音, 量词填写
- 高级翻译：英中输入, 中英输入
- 高级听写：听写短句, 听写段落
- 综合运用：看图选词全部4种变体

## 十、关键指标与健康度监控

### 10.1 用户学习健康指标

| 指标 | 健康范围 | 预警值 | 干预策略 |
|------|---------|-------|---------|
| 每日学习时长 | 10-30分钟 | <5分钟 | 推送鼓励通知 |
| 节测试通过率 | ≥70% | <50% | 建议降速复习 |
| 复习完成率 | ≥80% | <50% | 暂停新课，强制复习 |
| 错题堆积量 | <50题Red | >100题Red | 开启错题消化模式 |
| 连续学习天数 | ≥3天/周 | 断学>3天 | 推送召回通知+降难度 |

| 指标 | 用途 |
|------|------|
| 各知识点平均正确率 | 发现内容设计问题（正确率<30%可能是题目有bug） |

| 错题从Red到Green的平均耗时 | 评估复习算法效果 |
