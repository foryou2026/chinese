# 课程体系总体架构规划

> 东南亚垂直行业汉语 App · 主题×阶段矩阵 · HSK 1-6 封顶
> 数据标准：HSK 3.0（2026修订版）官方大纲

---

## 一、核心设计原则

### 1.1 开放学习，零门槛

用户可以**自由查看和学习任何阶段的任何内容**，系统仅做学习进度的客观记录，不设强制解锁门槛。测试结果作为能力评估参考，不阻断学习路径。

### 1.2 HSK 6 级封顶

产品砍掉 HSK 7-9 级，最高至 6 级封顶。面向东南亚务工、电商、日常场景的实用性需求，6 级已覆盖全部目标用户的能力天花板。

### 1.3 主题分流 + 公共地基

Stage 1-2 为所有主题共享的公共基础课（Foundation），Stage 3-6 按行业/场景主题分流，每个主题独立的词汇体系和场景设计。

### 1.4 两套内容标准

| 阶段 | 内容标准 | 核心策略 |
|------|---------|---------|
| Stage 1-2 | **100% HSK 官方大纲** | 利用 HSK 成熟的标准化数据，零基础教学质量绝对不翻车，最大化降低研发成本 |
| Stage 3-6 | **HSK 语法骨架 + 行业血肉** | 语法严格复制 HSK 3-6 级递进难度，词汇/例句 100% 替换为垂直行业场景 |

## 四、内容编写标准（研发准则）

### 4.1 Stage 1-2：通用地基阶段

**标准**：100% 采用 HSK 官方大纲的词汇、语法和语音体系。

| 维度 | 要求 |
|------|------|
| 词汇 | 严格使用 HSK 1/2 级官方词表，不增不减 |
| 语法 | 严格使用 HSK 1/2 级语法点列表 |
| 语音 | 标准普通话声韵母 + 四声调 + 变调规则 |
| 场景 | HSK 标准教材场景（问候、家庭、购物、出行等） |
| 例句 | 优先使用 HSK 官方例句，补充时严格控制在本级词汇范围内 |

**目的**：利用 HSK 极其成熟和标准化的开源数据，确保零基础阶段教学质量绝对不翻车，同时最大化降低研发成本。

### 4.2 Stage 3-6：行业垂直阶段

**标准**："语法用 HSK 的骨架，词汇/例句用主题的血肉"

#### 裁剪原则

| 维度 | 规则 |
|------|------|
| **语法** | 严格复制 HSK 3-6 级的语法递进难度，但剃除不符合现代职场/电商口语的陈旧语法 |
| **词汇** | 100% 脱离 HSK 官方词表。电商主题只允许电商词汇，工厂主题只允许工厂词汇 |
| **词汇量对齐** | 非HSK主题每级词汇量须与HSK官方同级词汇量一致（如Stage 3 = 500词，Stage 4 = 1019词） |
| **场景** | 100% 行业实战场景。电商=直播/客服/供应链/对账；工厂=生产/质检/安全/设备 |
| **例句** | 每句必须同时包含指定语法点 + 行业词汇，不允许出现"教材感"的通用句 |

#### 各行业主题词汇边界（硬性红线）

| 主题 | 允许的词汇场景 | 禁止出现的内容 |
|------|-------------|-------------|
| 日常 (Daily Life) | 餐饮、出行、租房、就医、社交、旅游、娱乐、银行、新闻 | 电商术语、工厂术语 |
| 电商 (E-commerce) | 直播带货、客服话术、供应链、物流对账、平台运营、数据分析 | 工厂术语、HSK 标准课文 |
| 工厂 (Factory) | 生产管理、安全规范、质量检查、设备操作、仓储盘点、人力管理 | 电商术语、HSK 标准课文 |
| HSK (Exam Prep) | HSK 3-6 官方大纲全部内容 + 应试技巧 | 行业专属术语 |

#### 查重机制

| 规则 | 说明 |
|------|------|
| Stage 3-6 非HSK主题 vs Stage 1-2 | **零重复**：行业主题词汇不得与公共基础阶段任何词汇重复 |
| 同主题跨阶段 | **零重复**：同一主题的 Stage 3 和 Stage 4 之间不得有词汇重复 |
| 同阶段跨主题 | **允许交叉**：同一阶段的电商/工厂/日常之间允许合理交叉重复 |
| HSK主题 | **严格官方**：100% 使用官方词表，不存在查重问题 |

## 五、AI 课件生成控制公式（Stage 3-6 专用）

> 防止 AI 批量生成课件时质量失控或难度超纲

### 5.1 三重限定指令

```
AI 课件生成限制逻辑：

1. 【限定语法】必须且仅能选用 [指定HSK级别] 内的一个特定语法点
   示例：HSK 3 级的"除了……以外"

2. 【行业词汇注入】必须且仅能带入 [2-3个对应行业核心词]
   示例：电商主题注入"包邮、爆款、秒杀"

3. 【难度严控】除了上述指定的语法点和行业词外，
   整个对话/文本中严禁出现超过当前级别难度的生僻汉字和词汇
```

### 5.2 各主题生成示例

**电商主题 · Stage 3 · HSK 3 语法"除了……以外"**：

```
语法锚点：除了……以外
注入行业词：包邮、爆款
生成结果：
  "除了这款爆款以外，其他商品都不包邮。"
  "除了周末以外，我们每天都有爆款活动。"
违规示例（禁止）：
  ✗ "除了这款爆款以外，其他SKU的ROI都不达标。" ← "SKU"和"ROI"超纲
```

**工厂主题 · Stage 4 · HSK 4 语法"不管……都……"**：

```
语法锚点：不管……都……
注入行业词：质检、合格率
生成结果：
  "不管订单多大，质检都不能省。"
  "不管用什么方法，合格率都必须达到98%。"
违规示例（禁止）：
  ✗ "不管PDCA循环怎么优化，CPK值都上不去。" ← "PDCA"和"CPK"超纲
```

### 5.3 质量校验清单

每条 AI 生成内容必须通过以下校验：

| # | 校验项 | 通过标准 |
|---|-------|---------|
| 1 | 语法点来源 | 属于指定 HSK 级别的语法列表 |
| 2 | 行业词数量 | 恰好 2-3 个，不多不少 |
| 3 | 行业词归属 | 属于指定主题的词汇边界内 |
| 4 | 超纲词检测 | 扫描全文，无超出当前级别的生僻字/词 |
| 5 | 场景真实性 | 对话/文本场景在真实行业中确实存在 |
| 6 | 口语自然度 | 符合现代职场/电商口语习惯，非书面腔 |
| 7 | 查重校验 | 词汇不与 Stage 1-2 及同主题前序阶段重复 |
