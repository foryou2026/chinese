> **落盘**：`docs/A00-meta/questions/L-<feature-id>-questions-round<N>.md`

---

## AI 必须遵守

1. **只读**：L 用户输入 + 本 feature R 基线 + B01 04-api-conventions + B02 全部 + 本 feature C02 IA（重点 04-pages.md、03-state-machines.md、01-feature-catalog.md） + 本 feature C03 页面交互 + 本 feature D01 D + `A00-01`、`A00-03`、`A00-04` + 本模板。
2. 第一轮回复必为本模板实例化。
3. 阻断 ≤ 10、优化 ≤ 20。
4. **本阶段同时负责路由**：必须将 C02 04-pages.md 的 page-id 反向映射为 URL；page-id 不得增删。

---

## 必盘点的 16 维

1. **路由：C02 04-pages.md 中所有 page-id 与 URL 一一对齐**（无遗漏、无增增、无冲突）
2. **状态转移接口覆盖：C02 03-state-machines.md 中每条 SM 转移都有接口承接**
3. 资源命名 vs 动作命名（REST 风格 vs RPC 风格）
4. 路径参数 vs 查询参数 vs body 参数 取舍
5. 操作的最小集（避免对 1 个资源出 6 个写接口）
6. 列表查询的筛选/排序/分页参数全集
7. 鉴权与权限要求（哪些角色能调，行级如何过滤）
8. 输入校验规则与错误码范围
9. 业务规则在哪一层校验（DB/Service/Controller）
10. 事务边界
11. 幂等：哪些写接口必须幂等、Idempotency-Key 要求
12. 长任务策略与状态查询接口
13. 限流与配额
14. 副作用（事件、邮件、日志、回调）的同步/异步
15. 第三方调用的失败兜底（mock 期 vs 真实期）
16. 响应字段裁剪（脱敏、按角色返回不同字段）
