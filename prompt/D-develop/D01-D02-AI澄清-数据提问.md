## AI 必须遵守

1. **只读**：D 用户输入 + 本 feature R 基线 + 本 feature C02 IA（重点 03-state-machines.md 作为状态枚举来源） + B01 03-database + B02 03-authz-data-model.md + `A00-01`、`A00-03`、`A00-04` + 本模板。
2. 第一轮回复必为本模板实例化。
3. 阻断 ≤ 10、优化 ≤ 20。
4. 表名/字段名先用业务术语，类型/索引在澄清里推荐。
5. **不要重画状态机**：状态机本身在 C02 已冻结，本阶段只问「如何营造状态字段」（枚举名、默认值、是否在 DB 层加 CHECK）。

---

## 必盘点的 12 维

1. 表边界（一个对象一张表 vs 合表 vs 拆表）
2. 主键策略（uuid/snowflake/自增）
3. 通用字段是否齐（时间、软删除、创建人）
4. 枚举字段的所有可能值与默认值
5. 一对多 / 多对多 / 自关联 关系
6. **状态字段枚举与 C02 03-state-machines.md 同名 SM 是否一致**（如不一致必须回 C02 修订，不能在 D 阶段改）
7. 计算/派生字段是存还是算
8. 索引（覆盖哪些查询、唯一索引、复合索引顺序）
9. 唯一约束（业务唯一键 vs 主键）
10. 字段级校验（长度、格式、范围、正则）
11. 历史 / 版本 / 软删除策略
12. 量级 / 分区 / 归档
