> **谁产出**：AI（信息架构师）
> **落盘**：`docs/A00-meta/questions/I-<feature-id>-questions-round<N>.md`

---

## AI 必须遵守

1. **只读**：I 用户输入 + 本 feature R 基线 + B02 全部 + B03、B04 索引 + `A00-01`、`A00-03`、`A00-04` + 本模板。
   > **严禁读**：docs/B01-architecture/ 与任何开发阶段（数据 / 接口 / 校验）产出（C 与 D 隔离；I 阶段不依赖 B01 架构与任何开发阶段产物）。
2. 第一轮回复必为本模板实例化。
3. 阻断 ≤ 10、优化 ≤ 18。
4. **本阶段产物只到“产品形态”**：功能模块、流程图、状态机、页面清单、导航结构。**不许**向用户提问 URL 路径、API endpoint、表字段、HTML 元素——那些是后续开发阶段（数据 / 接口）与 C03 阶段的事。

---

## 必盘点的 12 维（出题前每维都要摸过）

1. **功能模块拆分**：模块边界是否清晰、有无重叠、命名是否一致
2. **模块 × 角色**：每个模块对各角色是否可见 / 可写
3. **主流程的入口、分叉、终态**是否完整
4. **异常流程**（失败 / 超时 / 无权限 / 无数据 / 重复操作 / 并发）是否被覆盖
5. **状态机覆盖**：哪些业务对象需要状态机、状态枚举是否齐全、是否有终态、是否允许回退
6. **状态转移触发者**：谁能触发（角色 / 系统 / 定时 / 事件）、前置条件是否明确
7. **页面颗粒度**：列表 / 详情 / 编辑三件套是否统一形态、弹窗 vs 抽屉 vs 新页 决策依据
8. **页面 × 模块**：每个页面归属哪个 M-ID、每个 M-ID 是否都至少有一个页面承载
9. **空数据态独立页 vs 列表内空态**
10. **401/403/404/500 系统兜底页归属**
11. **导航层级与角色可见性**（菜单项以 page-id 表达，不出 URL）
12. **与 R 基线 §4 需求清单的覆盖矩阵**是否完整（每个 R-ID 都能落到某个功能/页面/状态转移上）
