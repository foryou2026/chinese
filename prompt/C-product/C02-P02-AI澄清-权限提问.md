# 09 · P02 AI 澄清：权限提问清单生成模板

> **阶段**：P 权限与角色
> **谁产出**：AI（安全/权限工程师）
> **落盘**：`docs/A00-meta/questions/P-questions-round<N>.md`

---

## AI 必须遵守

1. **只读**：P 用户输入 + B01 输出 + `A00-01`、`A00-03`、`A00-04` + 本模板。
2. 第一轮回复必为本模板实例化。
3. 阻断级 ≤ 10、优化级 ≤ 20。

---

## 必盘点的 10 维（去登录化版本）

> **不许**问登录方式 / 注册路径 / Token / 会话超时 / 多设备策略 / 验证码 / 第三方登录 —— 那些已在 B01 `09-auth-infra.md` 与各 surface 的 `<surface>-auth` feature 中定义。本阶段**只**盘 角色 + 授权 + 数据可见 + 合规。

1. 角色枚举与互斥/包含关系（必须与 C02-P01 §1 完全一致）
2. **角色 × surface 矩阵**（每个角色出现在哪些 surface；同一角色跨 surface 是否共享 user 实体）
3. 多角色共存策略（同一 user 可否同时持有多角色；如何切换）
4. 权限粒度（页面级 / 功能点级 / 字段级 / 行级）
5. 越权检查方位（前端守卫 / 后端中间件 / DB 行级 RLS / 多重）
6. 数据可见范围原则（owner-only / dept / org / cross-tenant 等）
7. 资源所有权传递（创建者转移、离职继承）
8. 审计日志（谁能看、保留多久、是否按 surface 拆分）
9. 数据合规（GDPR / 个保法 / 越南 / 出口管控等项目需要满足的法规）
10. 系统级硬约束（如"管理员不能删除自己""禁用账号登录提示"等需在 authz 层面落地的规则）
