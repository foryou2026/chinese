# 18 · R02 AI 澄清：需求提问清单生成模板

> **阶段**：R 需求分析
> **谁产出**：AI（需求分析师）
> **何时产出**：收到 C01-R01 用户输入 后第一轮回复，必为本格式。
> **落盘**：`docs/A00-meta/questions/R-<feature-id>-questions-round<N>.md`

---

## AI 必须遵守

1. **只读**：R 用户输入 + `A00-01`、`A00-03`、`A00-04` + 本模板。
   > **严禁读**：docs/B01-architecture/ 与任何开发阶段（数据 / 接口 / 校验）产出（C 与 D 隔离；R 阶段也不依赖 B01 架构）。
2. **只输出**：本模板的实例化 Markdown。
3. 所有问题必须有：编号、可选项、推荐、推荐理由、影响范围。
4. 阻断级 ≤ 10、优化级 ≤ 20。
5. 禁止 yes/no 之外没选项的开放题。

---

## 必盘点的 11 维（出题前每维都要摸过）

1. 用户类型与画像是否清晰、是否遗漏角色
2. 业务目标是否可量化、口径是否明确
3. 功能诉求是否有“名词式”需求（要拆成动作）
4. 业务场景叙述是否能明确评估价值、是否覆盖主要角色
5. 业务规则的边界值与冲突
6. 角色与权限的粗划分能否映射所有功能
7. 第三方集成的失败兼底（mock 期 vs 真实期）
8. 数据从哪来、归谁所有、可见范围
9. 国际化、时区、货币、日期格式
10. 设备与网络（PC/移动/弱网/离线）
11. 排除项是否真能排除（功能间是否有暗依赖）

> **不要提问“主流程分叉」、“状态机详细转移」、“每个页面上面有什么”这类问题——那是 C02 信息架构阶段的责任。R 阶段充分需求，结构化刚刚够用。**

## 出题质量自检

- [ ] 是否每条问题都标了"影响范围"？
- [ ] 阻断级 vs 优化级 是否分清？
- [ ] 是否避免了"你还想要什么？"这种空问题？
- [ ] 是否避免了一题多问（一题只问一件事）？
- [ ] 推荐项是否真的合理？
- [ ] 总数是否在限额内？
- [ ] 11 维盘点是否每维都过了？

任何一项 No → AI 自己重写，不要交给用户。

```markdown
- **推荐**：B. 不做离线，仅做弱网图片懒加载

（或）
- **答**：选 A，离线必须做。补充：仅离线最近 3 节课。
```

## 多轮规则
- 至多 3 轮
- round2 起每轮新问题 ≤ 5 个
- 3 轮仍未收敛 → 输出"R 用户输入不可用，需补写章节 X / Y"，球踢回用户
