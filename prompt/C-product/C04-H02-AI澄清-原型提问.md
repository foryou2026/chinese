> **谁产出**：AI（HTML 原型工程师）
> **落盘**：`docs/A00-meta/questions/H-<feature-id>-questions-round<N>.md`

## AI 必须遵守

1. **只读**：H 用户输入 + 本 feature C01-R 基线 + docs/B02-permissions/ + 本 feature C02-IA 全部 + 本 feature C03-N 全部页面规范 + docs/B03-ux/ + docs/B04-design/design-system/ + docs/B04-design/prototype-style/ + `A00-01` + `A00-03` + `A00-04` + 本模板。
2. **C/D 隔离铁律**：本阶段是产品设计阶段，**严禁**触碰任何后端实现细节——不读 docs/B01-architecture/、不假设接口签名、不假设数据库字段、不假设路由路径，也不向用户提问"这个调什么接口 / 字段叫什么 / URL 怎么写"。这些都是后续开发阶段的事，本阶段产出物里也不得出现。
3. 第一轮回复必为本模板实例化。
4. 阻断级 ≤ 8、优化级 ≤ 15。
5. 任何与 B03 决策、B04 token 冲突的备选不得出现。
6. **禁止**向用户提"是否分期 / 是否先做一部分 / 是否本期只做 P0" 这类问题——本阶段就是把 H01 §3 表里所有页面一次性出齐；如发现 H01 §3 与 C02 04-pages.md 不一致，按"是否一致"提问，不按"是否分期"提问。
7. 即使 H01 看似零歧义，也必须显式输出本轮自检结论；无问题时输出"本轮无澄清问题，进入 C04-H03"。

---

## 必盘点的 14 维

### A. 与上游一致性
1. H01 §1 / §3 列出的 page-id 是否与 C02 04-pages.md **完全一致**？多了 / 少了哪些？（多了→不在 IA 范围，应回 C02 处理；少了→询问是否漏写）
2. H01 §3 列出的 page-id 是否每一个都能在 docs/C03-pages/<feature-id>/ 找到对应 `<page-id>.md` + `<page-id>.scenarios.md`？缺哪些？
3. C02 03-state-machines.md 涉及的页面在原型里是否已要求表达不同状态下的差异（可见元素、可见操作、文案）？
4. B03 体验原则是否被违反（焦点态、动效时长、空状态文案语气、是否允许 emoji）？
5. B04 设计系统：本次需要的 token / 组件是否已存在？若不存在，是补 token（回 B04）还是绕开？**禁止**在 feature.css 重定义 token。
6. B02 角色清单与 H01 §1 主要角色是否一致？是否有页面需 forbidden 态而 H01 §7 未列入？

### B. 原型实现边界
7. 数据来源：mock-data.js 中 OP-ID 与 docs/C03-pages/<feature-id>/<page-id>.md §4 操作清单是否一一对应？字段名是否全部用业务语言（**禁用**数据库列名 / 后端字段名）？
8. 交互真实度：H01 §5 表单校验 / 列表分页 / 动效选择是否与 B03 一致？有冲突时以哪条为准？
9. 跳转：是否所有页面间跳转都用 `<page-id>.html` 相对路径锚？H01 是否出现真实 URL 路径需求（应拒绝并改为文件锚）？
10. 可访问性：键盘可达、`aria-*`、`prefers-reduced-motion` 是否本轮纳入？
11. 状态四件套（默认/加载/空/错）：H01 §3 标 P0 的页面是否全部要求 4 态？P1 页面默认态是否够用？
12. 移动端断点：375 / 768 / 1280 三档是否全做？
13. SEO / GEO：H01 §9 列出的公开页是否与 C03 各页 §11 一致？
14. 全局视觉诉求（H01 §6）若涉及 token 变更，是否已确认会回写 B04 源？
