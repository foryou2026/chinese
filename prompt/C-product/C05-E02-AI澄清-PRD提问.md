# 30 · E02 AI 澄清：PRD 提问清单生成模板

> **阶段**：E 产品需求文档（PRD）
> **谁产出**：AI（产品经理）
> **何时产出**：收到 `C05-E01` 用户输入后第一轮回复，必为本格式。
> **落盘**：`docs/A00-meta/questions/E-<feature-id>-questions-round<N>.md`

---

## AI 必须遵守

1. **只读**：E 用户输入 + 本 feature 的 R / I / N / H 全部冻结产物 + B01 / B02 / B03 / B04 全部冻结产物 + `A00-01`、`A00-03`、`A00-04` + 本模板。
   > **严禁读**：任何后续开发阶段产物（数据 / 接口 / 校验）。C05-E 阶段是产品设计收尾，PRD 只面向已冻结的“产品形态”，不允许引用或预言任何开发阶段产出。
2. **只输出**：本模板的实例化 Markdown。
3. 所有问题必须有：编号、可选项、推荐、推荐理由、影响范围。
4. 阻断级 ≤ 12、优化级 ≤ 20。
5. 一题只问一件事；禁止"你还想要什么"这种开放题。
6. 凡 E01 §4 中"是否已冻结"勾"否"的，必须升为阻断级或上游缺口告警，不得自行补全。
7. 凡 E01 中已明确填写的字段，AI 不得反问"你想要什么"，只能基于该字段进一步细化。
8. 本阶段不许新增需求。任何 H 阶段以外的"建议加一个功能"都属越界，应忽略。

---

## 必盘点的 14 维（出题前每维都摸一遍）

1. **PRD 读者画像与深度**：基于 §0 的"阅读对象 / 技术水平"，行文是否给字段表 / 是否含 ER 图 / 是否含 API 列表
2. **范围边界**：§3 三桶有无遗漏；模糊页面归哪桶
3. **模块拆分粒度**：§7 模块清单是否需要再合并 / 再拆；模块 ID 命名规范
4. **页面规格颗粒度**：每个页面是否单文件、是否嵌截图、是否分"展示态 / 操作态 / 异常态"
5. **用户故事 vs 功能描述的取舍**：是否用 "As a … I want … So that …" 形态、是否给验收标准
6. **业务规则呈现形式**：表格 / 编号列表 / 决策树；规则与上游 ID 的回链方式
7. **数据形态描述越界检查**：页面上出现的业务概念是否仅用业务语言描述（禁止出现表名 / 字段名 / SQL）
8. **接口 / 路由越界检查**：是否出现任何接口 endpoint / API-ID / URL 路径（禁止，都是后续开发阶段职责）
9. **角色与权限呈现**：复述 P 阶段权限矩阵 vs 简化版 vs 仅链接
10. **视觉与交互摘要**：是否嵌品牌色 / 字体 / 主组件截图，还是仅链接 X / S
11. **技术栈摘要**：是否给"接手第一周环境搭建步骤"；是否复述 A 全文
12. **已知缺陷与历史包袱**：§9 每一项是否需追问严重程度 / 复现步骤
13. **后续迭代方向**：§10 每一项是否需标注"已立项 / 仅构想 / 已暂停"
14. **PRD 文档治理**：版本号策略、变更记录写在哪、谁负责更新、与代码 commit 的关联方式

## 出题质量自检

- [ ] 14 维盘点是否每维都过了？
- [ ] 阻断级 ≤ 12、优化级 ≤ 20？
- [ ] 每条问题都标了"影响范围"？
- [ ] 每条问题都引用了 E01 字段或上游某节？
- [ ] 是否避免了"你想给谁看"这种 §0 已答过的伪问题？
- [ ] D 节是否每条都标"不接受请变 Q"？
- [ ] D 节有没有把 E01 没填的字段偷偷声明掉（违规：必须升 Q）？

任何一项 No → AI 自己重写，不要交给用户。

---

## 多轮规则

- 至多 3 轮
- round2 起每轮新问题 ≤ 5 个
- 3 轮仍未收敛 → 输出"E 用户输入不可用，需补写章节 X / Y"，球踢回用户
