# B02-02 AI 输出：体验定调与设计系统

> **阶段**：B02 体验与设计系统
> **谁产出**：AI（体验总监 + 设计系统工程师）
> **输入**：用户填写的 `B02-01-用户输入-体验与设计.md` + 已产出的 B01 架构规范

---

## 触发提示词

```
我已填完体验与设计偏好。请按 /prompt/B-foundation/B02-02-AI输出-体验与设计系统.md 的多文件结构输出。
所有形容词必须配 1 条可验证维度 + 1 条参考 + 1 条反例。
同时产出两份产物：
  1. design-system/ -- markdown 规范（token/布局/导航/状态色/组件/交互/响应式与暗黑）
  2. prototype-style/ -- 可运行 CSS/JS 资产
两份产物 token 完全同源。
每个组件必须出 6 态（默认/hover/focus/active/disabled/loading）+ 异常态。
单文件 <= 1200 行。未决项写入 99-open-questions.md。
```

---

## AI 执行规则

1. 体验定调（01-06）以 B02-01 用户输入的体验部分为输入。
2. 设计系统（design-system/ + prototype-style/）以用户输入的视觉部分 + 体验定调产出为输入。
3. "由 AI 定"的项必须给出具体值 + 一句话理由。
4. 禁止出现"现代/简洁/高端/大气/年轻化/科技感"。
5. `prototype-style/tokens.css` 中每个 CSS 变量必须与 `design-system/01-tokens.md` 逐字一致。
6. 输出精简，不写教程式解释。每节聚焦核心产出。

## 可运行资产：prototype-style/README.md

```markdown
# 原型样式包 · 使用说明

## 引入方式

```html
<link rel="stylesheet" href="<相对路径>/prototype-style/app.css">
<script defer src="<相对路径>/prototype-style/app.js"></script>
<script>proto.bootstrap();</script>
```

页面级微调样式/微交互一律内联在 HTML 的 <style>/<script> 中。

## 全局命名空间

- CSS 变量：`--color-* / --space-* / --text-* / --radius-* / --shadow-* / --motion-* / --z-* / --font-*`
- JS 全局：`window.proto`（bootstrap / switchTheme / switchAccent / toast / modal / drawer / dropdown）

## DOM 三轴标记

| 轴 | 取值 | DOM |
|----|------|-----|
| 模式 | light / dark / auto | `<html data-mode="...">` |
| 主题色 | accent-1 / accent-2 / ... | `<html data-accent="...">` |
| 密度 | default / compact / elder | `<html data-density="...">` |

## 红线
1. 严禁在页面中重定义任何 token，只可消费
2. 严禁写死 hex / px
3. 所有可聚焦元素必须可见焦点环
4. 必须支持 prefers-reduced-motion: reduce
```
