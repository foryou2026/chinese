## 2.5 应用端清单（surface inventory · 关键全局决策）

> 一个项目可能同时存在多个**应用端**（surface）：管理端、商家端、移动应用、官网、开放 API……每个端有独立用户群、独立鉴权域、可能独立部署。**先把端列清楚**，下游 C/D 阶段才知道按哪些端去拆。
>
> **单端项目**：只填 1 行，下游一切按原方式跑，无任何额外心智负担。
> **多端项目**：列出所有端，下游 C02~C05 按 surface 子目录产出；C01 R 与 D01 D 跨端共享一份。

| surface-id | 显示名 | 用户群 | 鉴权方式（粗，仅类型）| 部署形态 | 是否本期上线 |
|------------|-------|-------|--------------------|---------|------------|
| `app` | 用户端 | 终端用户 | 账号密码 + 微信登录 | Web SPA + PWA | ✅ |
| `admin` | 管理后台 | 内部运营 | 账号密码 + 强 MFA | Web SPA（内网）| ✅ |
| `merchant` | 商家端 | 入驻商家 | 账号密码 | Web SPA | 二期 |
| `open` | 开放 API | 第三方系统 | API Key / OAuth2 | 仅后端 | 二期 |
