## 二、命名约定

| 对象 | 规则 | 示例 |
|------|------|------|
| feature-id | 全小写，短横分隔 | `order-mgmt` |
| endpoint-id | `<method>-<resource>` | `get-orders`、`post-orders` |
| page-id | `P-<seq3>` | `P-012` |
| scope | `public` / `app` / `admin` | `app-home.html` |
| 文件名小写 + 短横 | 不要空格、不要中文 | `01-tech-stack.md` |
| 索引 `00-index.md` | 每个有多文件的子目录都要有 | — |
| 待确认 `99-open-questions.md` | 每个子目录都要有 | — |

## 三、AI 落盘行为约束

每次 AI 输出最终文件时必须：

1. **声明落盘路径**：在文件头 `---` 之前一行写：
   ```
   <!-- TARGET-PATH: docs/40-data/order-mgmt/01-tables.md -->
   ```
2. **多文件输出**：若一次产出多个文件，逐个用 `<!-- TARGET-PATH: ... -->` 标注；用户按标注切分保存。
3. **拆分原则**：单文件 ≤ 1200 行；超出按本规划的子目录结构拆。
4. **更新索引**：拆分后必须同时更新对应 `00-index.md`，列出所有子文件 + 一句话职责。
5. **移交问题**：未决问题写入对应子目录的 `99-open-questions.md`，不是塞在正文。

---

## 四、变更与冻结

- 单文件冻结：在文件头 `冻结状态` 改为 `已冻结`。
- 子目录冻结：在该子目录 `00-index.md` 顶部写 `冻结状态: 已冻结 · YYYY-MM-DD · 签字: <name>`。
- 变更记录：所有冻结/解冻写入 `docs/00-meta/changelog.md`。
