## 5. 业务流程（mermaid）

### 5.1 主流程：新用户从注册到首次完成一节课
```mermaid
flowchart LR
  A1[打开网站] --> A2[选母语]
  A2 --> A3[邮箱注册]
  A3 --> A4{收到验证邮件?}
  A4 -->|是| A5[点击验证]
  A4 -.->|否(60s)| A6[重发按钮可点]
  A5 --> A7[登录]
  A7 --> A8{首次?}
  A8 -->|是| B1[水平测试 5 题]
  A8 -->|否| C1[今日课表]
  B1 --> B2[出等级 N1~N5]
  B2 --> C1
  C1 --> C2[选第一节课]
  C2 --> C3[学习]
  C3 --> C4[打卡]
  C4 --> C5[激励文案+徽章]
```

### 5.2 异常流程：支付失败
```mermaid
flowchart LR
  P1[选套餐] --> P2[拉起支付]
  P2 -.->|3s 超时| P3[提示重试]
  P2 -.->|渠道返回失败| P4[显示错误码与客服入口]
  P3 --> P2
  P4 --> P5[订单状态=支付失败]
```

> 至少 1 张主流程 + 1 张异常路径图。如有复杂状态机，加 stateDiagram-v2。

### 5.3 状态机（如有，例：订单）
```mermaid
stateDiagram-v2
  [*] --> 待支付
  待支付 --> 已支付: 支付成功
  待支付 --> 已取消: 用户取消/超时15m
  已支付 --> 已发货: 后台发货
  已发货 --> 已完成: 用户确认/自动7d
  已完成 --> [*]
  已取消 --> [*]
```
