## 二、本框架在 G 阶段补的"澄清环节"（关键）

老模板缺的就是"AI 拿到用户输入后必须先问、再写"这一环。本文件补齐。

每个 G 子阶段的标准三步走：

```
Step 1  你按 G<X>-用户 模板填写 → 落盘 prompt/<proj>/G<X>-用户.md
Step 2  AI 按本文件 §三 的"G<X>-questions 模板"先出澄清问题清单
        落盘 grules/<proj>/G<X>-架构与技术规范/_questions/round1.md
Step 3  你回答澄清后，AI 才按 G<X>-AI 模板出最终规范
        落盘 grules/<proj>/G<X>-架构与技术规范/00-index.md + 子文件
```

> 闸门：每个 Gx 输出文件的 `99-待确认问题清单.md` 必须为空才能进入下一个 G。

## 四、各子阶段必盘点维度（AI 出题前自查）

### G1 必盘点 14 维
1. 前后端语言版本（含包管理器）
2. 单/多仓库结构、模块边界
3. 数据库引擎、托管方式（云 / 自托管 / 本地）
4. 数据访问层（ORM / Query builder / 原生）
5. API 风格（REST / GraphQL / RPC），版本策略
6. 鉴权与会话（JWT/Session/OAuth）（与 G3 关联，先点结论）
7. 任务队列 / 定时任务
8. 缓存策略 / Redis 用途
9. 日志、监控、追踪
10. 部署形态（Docker / K8s / 裸机）、CI/CD
11. 多语言、时区、货币
12. 测试金字塔（单测/集成/E2E）边界
13. 第三方集成清单（含 mock 时机）
14. 文件上限、命名、目录约束

### G2 必盘点 12 维
1. 设计 Token 来源（自定义 / 现有 UI Kit）
2. 色彩系统（语义色 + 中性灰阶分级）
3. 字体（自托管 / CDN）、字号阶
4. 间距阶（4px / 8px 基线）
5. 圆角、阴影、动效曲线
6. 布局 grid（列数、最大宽度、断点）
7. 组件库选型 vs 自研
8. 状态色（success/warning/error/info）规范
9. 暗色模式策略
10. 国际化对 UI 的影响（长文案、阿拉伯方向）
11. 移动端导航形式与触控热区
12. 图标库与插画风格

### G3 必盘点 10 维
1. 角色枚举与互斥/包含关系
2. 权限粒度（页面级 / 功能点级 / 字段级）
3. 多角色共存（一个用户多个角色）
4. 注册路径（自助 / 邀请 / 后台开通）
5. 登录方式（邮箱 / 手机 / 第三方），mock 策略
6. Token 类型与刷新策略
7. 会话超时、踢人、单设备/多设备
8. 越权检查的位置（前端 / 中间件 / DB RLS）
9. 审计日志（谁能看、保留多久）
10. 注销与数据合规（GDPR/越南/泰国类似法规）

## 六、闸门 G-G 判定

- G1/G2/G3 三套 AI 输出的 `99-待确认问题清单.md` 都为空
- G2 的设计 Token 与 G1 的前端栈兼容（如 G1 选 React，则 G2 Token 可被 CSS 变量消费）
- G3 的角色枚举与 R3 §3 角色清单一致（不可多不可少）
- 三份输出都签字归档为 `已冻结 v1`

> 通过后才允许进 F 阶段。
