## 核心铁律（请背下来）

1. **三件套**：每一步 = 用户输入模板 + AI 澄清提问 + AI 输出。AI 不澄清直接出报告 = 立刻打回。
2. **小盒子上下文**：每一步只塞当前阶段需要的上游文件，禁止"以防万一全塞"。
3. **冻结才能往下走**：上游不签字冻结，下游不允许动笔。
4. **单文件 ≤ 1200 行**：超出强拆。
5. **角色独立**：R/P/G/F/V 用独立对话窗口，避免角色串场。

```

F → function/<proj>/<feature>/{F1,F2,F3}-*.md
V → function/<proj>/<feature>/_v/{V1,V2,V3}-*.md
```

## 启动新项目（一句话流程）

```
建目录 → 填 R1 → AI 澄清 → 出 baseline → 冻结 →
填 P1 → AI 澄清 → 出 IA/pages → AI 出 HTML → 迭代 → 冻结 →
按 30 跑 G1/G2/G3（每个都"填表→澄清→出规范"）→ 冻结 →
按 40 逐 feature 跑 F1/F2/F3（每个都"填表→澄清→出规范"）→ 冻结 →
按 50 跑 V1/V2/V3 → 冻结 → 进入编码
```

> 任何一处偷懒，后果由偷懒的人承担。框架不背锅。
