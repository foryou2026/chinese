## 原型规范

- **导航目录**（各级 index.html）仅做快捷跳转入口，按系统→模块→功能三层组织
- **原型页面**在系统根目录下，形成系统级闭环，页面间自由互联
- 页面与功能多对多：一个页面可服务多个功能，一个功能可关联多个页面
- 模拟真实交互：弹框、新建/编辑、表单验证、动态字段联动显隐，均需实现
- 不接真实数据和接口，使用 Mock 数据
- 增量扩展：新功能 = 新增导航条目 + 新增/修改原型页面

## 命名约定

| 对象 | 规则 | 示例 |
|------|------|------|
| system ID | 全小写唯一 | `app`、`admin` |
| 模块 ID | 小写短横 | `auth`、`course` |
| 功能 ID | 小写短横 | `account-entry` |
| 需求 ID | `R-<module>-<seq3>` | `R-auth-007` |
| 模块清单 ID | `M-<module>-<seq3>` | `M-auth-001` |
| 流程 ID | `FL-<module>-<seq3>` | `FL-auth-001` |
| 状态机 ID | `SM-<module>-<seq3>` | `SM-auth-001` |
| 业务规则 ID | `BR-<module>-<seq3>` | `BR-auth-001` |
| 页面 ID | `P-<system>-<module>-<seq3>` | `P-app-auth-012` |
| 接口 ID | `API-<system>-<module>-<verb>-<noun>` | `API-app-auth-create-session` |
| 角色 ID | `ROLE-<NAME>` | `ROLE-USER` |
| 表名/字段 | 按 B01 DB 规范 | `order_items` |

> 页面 ID / 接口 ID 带 system 前缀防撞号；M-ID / FL-ID / SM-ID / BR-ID 不带 system 前缀（随 C02/D01 文件归属系统隔离）。

## Mermaid 规范

- `mermaid` 代码块，禁图片
- 主路径 `flowchart LR`；状态机 `stateDiagram-v2`；时序 `sequenceDiagram`；ER `erDiagram`
- 节点 ID 大写+数字，名称写 `[]` 内；正常 `-->`，异常 `-.->`
- 每张图后补文字说明
- 必须适配Excalidraw的规范，保证导入时不报错

## 输出效率（Token 节约）

> 质量是绝对前提，在不损失信息密度和规范完整性的条件下，最大限度压缩输出体积。

| 规则 | 说明 |
|------|------|
| 表格优先 | 能用表格表达的信息禁止写成段落 |
| 零废话 | 禁止过渡句（"下面我们来看"）、总结句（"综上所述"）、自我说明（"本节定义了…"）|
| 一行一意 | 每条描述一句话封顶，不追加同义复述 |
| 空则写"无" | 章节无内容写"无"一字，禁止填充性段落 |
| 不复读上游 | 上游已冻结的定义直接引用 ID，不抄原文 |
| 禁止装饰 | 不加 emoji、不用感叹号、不写鼓励性语句 |
| 紧凑 mermaid | 节点名≤8 字，边标签≤6 字，无纯装饰节点 |
| 理由一句话 | "由 AI 推荐"选型理由不超过 15 字 |

## 大文件拆分规则

当单个 `.md` 文件输出超过 1200 行时，AI 必须：

1. 将 `xxx.md` 改为 `xxx/` 同名文件夹
2. 在文件夹内拆分为多个子文件，每个子文件 ≤ 1200 行
3. 文件夹内必须包含 `00-index.md` 作为索引

> 禁止为了控制行数而降低内容质量或省略必要信息。优先级：质量 > 精简 > 行数限制。行数限制通过拆分解决，不通过删减解决。

## 严禁

1. 擅自技术决策（应记入 §99）
2. 一次输出多阶段文件
3. 图片代替 mermaid
4. 越界（需求阶段写表结构等）
5. B 未冻结时启动 C
6. 按系统阶段跨系统引用
7. 共享产出放到系统目录
8. 在项目根目录直接放置源代码——所有开发代码必须置于 `system/` 目录下，根目录仅保留 `docs/`、`C03-prototype/`、`system/` 及项目配置文件

## 形容词具象化

任何风格/体验类形容词必须配：≥1 参照（产品名+借鉴点）、≥1 反例、≥1 可验证维度（如圆角≤6px、动效<200ms）。否则无效。
