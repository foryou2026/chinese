# A00-03 · 通用约定

> 所有阶段共用。AI 在任何阶段开口前必须先读本文件。

---

## AI 输出文件通用骨架

```markdown
# <文件名>

> **阶段**：<B01-A / B02-XS / C01-RP / C02-IN / C03-H / D01-D / D02-L>
> **角色**：<阶段对应角色>
> **归属**：<共享(_shared) / 按系统(<system-id>)>
> **系统**：<system-id>（共享阶段写「全系统」；B 层写「全局」）
> **模块**：<模块 ID>（B 层写「全局」）
> **功能**：<功能 ID>（B 层写「全局」）
> **上游依赖**：<实际读取的上游文件路径>
> **冻结状态**：未冻结 / 已冻结

---

## 0. 摘要（3-5 行）

## 1. 正文 ...

## 99. 待确认问题
- [ ] 编号：<问题>（影响：<下游文件>）
```

> 99 节为空 = 可冻结；非空 = 必须补答。

## Mermaid 规范

- 一律用 `mermaid` 代码块，禁止图片
- 主路径 `flowchart LR`；状态机 `stateDiagram-v2`；时序 `sequenceDiagram`；ER `erDiagram`
- 节点 ID 用大写字母+数字，可读名称写 `[]` 内
- 正常路径 `-->`，异常路径 `-.->`
- 每张图后补文字说明

---

## 命名约定

| 对象 | 规则 | 示例 |
|------|------|------|
| system ID | 全小写，项目内唯一 | `app`、`admin`、`merchant` |
| 模块 ID | 全小写短横分隔 | `auth`、`course`、`order` |
| 功能 ID | 全小写短横分隔 | `account-entry`、`learning` |
| 需求 ID | `R-<模块>-<seq3>` | `R-auth-007` |
| 模块 ID | `M-<模块>-<seq3>` | `M-auth-012` |
| 状态机 ID | `SM-<模块>-<seq2>` | `SM-auth-01` |
| 页面 ID | `P-<system>-<模块>-<seq3>` | `P-app-auth-012` |
| 接口 ID | `API-<system>-<模块>-<verb>-<noun>` | `API-app-auth-create-session` |
| 角色 ID | `ROLE-<NAME>` | `ROLE-USER`、`ROLE-ADMIN` |
| 表名/字段 | 按 B01 DB 规范（通常 `snake_case`） | `order_items` |

> 页面 ID 与接口 ID 必须带 system 前缀，确保跨系统不撞号。需求 ID 不带 system 前缀（需求是共享的）。

## 引用规则

- 仓内文件用相对路径 markdown 链接
- 同文件内章节：`见本文 §3.2`
- 上游冻结文件必须出现在头部「上游依赖」字段
- **跨系统引用禁止**：按系统独立阶段（C02/C03/D02）不得引用其他系统的同阶段产出；可引用共享阶段（C01/D01）
- 跨功能引用：原则上不允许，如确需先沉淀到共享层

## AI 自检 Checklist

每次输出前内部跑一遍，任何一项 No 则重写：

- [ ] 只读了系统消息列出的上下文？
- [ ] 输出文件头部元信息齐全（含系统、模块、功能）？
- [ ] 落盘路径正确（共享→`_shared/`；按系统→`<system>/`）？
- [ ] 有 `99 待确认问题` 一节？
- [ ] 未越界做本角色不该做的事？
- [ ] 文件未超 1200 行？
- [ ] 未塞入用户没要求的东西？
- [ ] 页面 ID / 接口 ID 带了 system 前缀？

---

## 严禁项

1. 擅自做技术决策（应记入 99 节）
2. 一次输出多份不同阶段的文件
3. 把上下文窗口塞满整个仓库
4. 用图片代替 mermaid
5. 越界（需求阶段写表结构、数据阶段写接口等）
6. B 层未冻结时启动 C 循环
7. 按系统独立阶段跨系统引用同阶段产出
8. 共享阶段产出放到系统目录下

---

## 形容词具象化原则

任何「风格/体验/感觉」类形容词必须配套：
- >= 1 个参照（产品名+截图链接+借鉴点）
- >= 1 个反例
- >= 1 个可验证维度（如：圆角<=6px、动效<200ms、留白比>=30%）

否则该形容词视为无效。
