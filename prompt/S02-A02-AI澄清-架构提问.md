# 10 · A02 AI 澄清：架构提问清单生成模板

> **阶段**：A 技术架构
> **谁产出**：AI（架构师）
> **何时产出**：收到 `13-A 用户输入` 后第一轮回复。
> **落盘**：`docs/S00-meta/questions/A-questions-round<N>.md`

---

## AI 必须遵守

1. **只读**：A 用户输入 + R 基线 + `S00-01`、`S00-03`、`S00-04` + 本模板。
2. 第一轮回复必为本模板实例化。
3. 阻断级 ≤ 10、优化级 ≤ 20。
4. 每条问题必有：可选项、推荐、推荐理由、影响范围。

---

## 必盘点的 14 维（出题前每维都要摸过）

1. 前后端语言版本与包管理器
2. 仓库结构、模块边界
3. 数据库引擎、托管方式（云 / 自托管 / 本地）
4. 数据访问层（ORM / Query builder / 原生 SQL）
5. API 风格（REST / GraphQL / RPC）与版本策略
6. 鉴权与会话基础选型（JWT/Session/OAuth）——细节留 P
7. 任务队列 / 定时任务
8. 缓存策略 / Redis 用途
9. 日志、监控、追踪
10. 部署形态、CI/CD
11. 多语言、时区、货币
12. 测试金字塔（单测/集成/E2E）边界
13. 第三方集成清单与 mock 时机
14. 文件上限、命名、目录约束
