## AI 必须遵守

1. **只读**：E 用户输入 + R 基线 + A 输出（前端栈 / i18n / SSR / 部署节）+ S 输出（品牌色 / 字体 / OG 资产）+ I 输出（路由 + 页面清单）+ D 输出（内容主表 / 知识点表）+ `S00-01`、`S00-03`（重点 §八）、`S00-04` + 本模板。
2. 第一轮回复必为本模板实例化，不得直接出方案。
3. 阻断级 ≤ 12、优化级 ≤ 20。
4. 每个问题必须配：可选项、推荐、推荐理由（引用至少一个 2025–2026 年实证数据点或一条上游约束）、影响范围（点名将影响 E03 的哪个文件）。
5. 严禁出现"你想要 SEO 还是 GEO"这种伪问题——SEO + GEO 必须双轮，问题只问"两者中哪一项的优先级 / 节奏 / 边界"。
6. 严禁绑定单一 SaaS（Ahrefs / SEMrush / Profound 等），所有外部工具一律给 ≥2 个候选 + 推荐。

---

## 必盘点的 14 维（每维至少 1 题，缺一不可）

1. **市场分层与 hreflang 策略**：P0 市场清单是否齐、locale 与 URL 结构是否吻合 A 阶段 i18n 决策
2. **关键词矩阵颗粒度**：按市场 / 语言 / 意图（信息 / 导航 / 交易 / GEO 引用）切到几层；是否有"AI 引用专用"长尾
3. **技术 SEO 地基补丁**：A 阶段 SSR / sitemap / robots / canonical / Core Web Vitals 是否全绿；缺哪几项需要先补
4. **Schema.org 覆盖**：要给哪几类页加 JSON-LD（Course / Article / FAQ / VideoObject / BreadcrumbList / Organization …），优先级
5. **/llms.txt 与 AI 抓取协议**：是否启用、内容索引粒度、是否对接 ChatGPT / Perplexity / Gemini 单独的 user-agent 策略
6. **BLUF + 语义分块写作规范**：哪些资产类型按 BLUF 改造（首句结论、表格 / 编号、明确时戳）；改造节奏
7. **Programmatic SEO 防爆**：单类目 URL 量、首批上线量、收录监测阈值、noindex 兜底策略
8. **内链架构**：枢纽页（Hub）/ 主题集群（Topic Cluster）/ 知识点之间的双向链接深度
9. **被引平台优先级**：Reddit / Wikipedia / YouTube / TikTok 各市场起跑顺序、账号矩阵 vs 单一账号
10. **AI 引擎差异化策略**：ChatGPT vs Perplexity vs Gemini vs 国产引擎，内容侧重与监测方法
11. **实体协同（Entity Harmonization）**：品牌名 / 创始人 / 知识图谱节点在 Wikipedia / Wikidata / Crunchbase / 各社交资料的一致性
12. **Freshness 节奏**：每类资产的更新周期、变更触发的内链 / sitemap / Schema 重抓
13. **监测栈**：GSC + GA4 + Server Logs + AI 引用追踪（自研 vs SaaS）+ 周报模版
14. **90 天 Sprint 与责任矩阵**：D0-D14 / D15-D45 / D46-D90 三阶段，每阶段交付物与验收标准
