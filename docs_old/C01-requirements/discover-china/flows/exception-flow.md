# `discover-china` · 异常流程（业务叙述）

> 流程图与详细状态机详见 [../../../C03-ia/discover-china/](../../../C03-ia/discover-china/)。本文仅以业务语言描述用户场景。

> **覆盖 R-ID**：R-discover-china-001 · 004 · 010 · 013 · 014 · 015 · 017 · 019 · 020

---

## 1. 未登录访问 04-12 类目

1. 访客点击 04-12 任一类目卡片。
2. 系统弹层提示"登录后继续"，主按钮跳到登录页并保留回跳目标。
3. 登录成功后回到该类目。

## 2. TTS 合成失败

1. 学员点击单句播放但合成请求失败。
2. UI 不阻塞，给出"稍后再试"提示。
3. 后台按重试策略再次合成；成功后下次点击命中缓存。

## 3. 多管理员并发编辑

1. 两位运营同时打开同一篇文章。
2. 较晚保存的一方触发"后写覆盖 + 提示"，告知是谁覆盖了较早的版本。
3. 系统不阻塞保存。

## 4. 危险操作二次确认

1. 运营对一篇文章执行删除 / 下架 / 大批量修改。
2. 系统弹层要求二次确认；用户取消则不执行。
3. 确认后执行；下架会同步清空所有用户对该文章的阅读进度。

## 5. 未保存离开

1. 运营在文章 / 句子编辑表单上有未保存改动。
2. 用户尝试路由切换或关闭页面。
3. 系统弹层拦截，请用户选择"继续编辑"或"放弃修改"。

## 6. 5 语未齐备发布

1. 运营在文章列表点击"发布"。
2. 系统检测到文章存在缺语字段。
3. 发布被拒绝，并给出缺失语种 / 句子的明确清单。

## 7. 软删后定期物理清理

1. 文章 / 句子被软删后进入回收窗口。
2. 软删后 30 天，后台脚本物理清理。
3. 类目永不进入清理窗口。
