# `auth` · app 端 · 主流程（业务叙述）

> 流程图与详细状态机详见 [../../../C03-ia/auth/](../../../C03-ia/auth/)。本文仅以业务语言描述用户场景。

> **覆盖 R-ID**：R-auth-001 · 002 · 003 · 004 · 005 · 007 · 008 · 009 · 010 · 011 · 015

---

## 1. 邮箱注册

1. 用户进入登录页，切换到"邮箱注册"。
2. 用户输入邮箱与密码（需满足强度提示）并提交。
3. 系统反馈"已发送验证邮件，请到邮箱完成验证"。
4. 用户在邮箱中点击一次性验证链接。
5. 验证成功后回到 app 自动登入并进入首页。

## 2. Google 一键登录

1. 用户在登录页点击"使用 Google 登录"。
2. 系统跳转到 Google 帐户授权页。
3. 用户在 Google 完成授权后回到 app。
4. 首次回调即建账户并登入；已有账户则直接登入。

## 3. 邮箱 + 密码登录

1. 用户在登录页输入邮箱与密码并提交。
2. 系统先做节流与禁用检查；通过后校验密码。
3. 校验通过即建立登录态并进入首页；若该设备使登入数超过上限，则最早设备被退出。

## 4. 找回密码

1. 用户在登录页点击"忘记密码"。
2. 用户输入邮箱并提交（系统回执统一文案，不暴露邮箱是否存在）。
3. 用户在邮箱内点击一次性链接（15 分钟有效）。
4. 用户设置新密码并提交。
5. 设置成功后自动登入，并使其它已登设备的登录态失效。

## 5. 修改密码 / 退出

1. 用户在个人中心进入"修改密码"。
2. 用户输入旧密、新密并提交；可勾选"全部设备退出"。
3. 修改成功后给出提示；若勾选"全部设备退出"，所有设备 30 秒内被踢回登录页。
4. 用户也可在头像菜单直接选择"本设备退出"或"全部设备退出"。
