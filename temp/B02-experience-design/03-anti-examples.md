## 0. 摘要

3 家反例，每家说清"它做了什么 / 我们绝不做什么 / 如何识别我们正在变成它"。

---

## 1. Duolingo（多邻国）

### 它的标志
- 巨大的绿色卡通猫头鹰吉祥物，几乎成为产品主视觉；
- 五彩斑斓的按钮：绿 / 蓝 / 紫 / 橘 / 红 / 粉同时出现；
- 喜欢用 emoji 当装饰；
- 每个微小操作都伴随大量音效与动画（雀跃猫头鹰、彩带、纸屑爆炸）；
- 错题 / 连胜中断时使用"心碎 / 罚款"等强情绪反馈。

### 我们绝不做
- ❌ 任何吉祥物当主视觉；如未来要有 mascot，**只能在 onboarding 一处出现，且体积 ≤ 64×64**；
- ❌ 主操作出现 ≥ 2 种颜色；唯一品牌色 = 红；
- ❌ 任何"彩带 / 礼花 / 纸屑"特效；
- ❌ 用强情绪文案羞辱用户（"哎呀！失败了！" "你又错了！"）；
- ❌ 在卡片 / 标题里放 emoji（emoji 仅在用户生成内容中允许）。

### 我们正在变成它的信号
- review 时听到"加个图标吧，看起来太严肃了" → 警惕；
- 任何一屏出现 ≥ 4 种颜色 → 警惕；
- PM / 设计师在原型里加了 mascot 想"提升萌感" → 直接打回。

## 2. HelloChinese / SuperChinese / Hanly（同类汉语学习 App）

### 它们的标志
- 主色调多为黄 / 橘 / 红混搭，组件土气；
- 卡片用实色背景 + 粗黑边框，无质感；
- 进度条使用渐变 + 闪烁特效；
- 操作密集：一屏上 6–8 个按钮、底部 5 个 tab、顶部 3 行；
- 中文教学场景里大量出现卡通拼音助手。

### 我们绝不做
- ❌ 黄 + 橘 + 红的"中国风混搭"配色；我们只有红 + 黑 + 白；
- ❌ 任何实色卡片背景；必须毛玻璃；
- ❌ 底部 tab > 3 个（应用端无底部 tab，统一顶部导航）；
- ❌ 渐变 / 闪烁进度条；进度条只用 `--brand` 纯色平滑过渡；
- ❌ "卡通拼音助手"角色。

### 我们正在变成它的信号
- 单屏控件数 > 8 个 → 必须做信息层次拆分；
- 出现 `bg-yellow-*` 或 `bg-orange-*` → 直接打回；
- 任何 GIF / lottie 的 mascot 跳舞 → 直接打回。

## 3. 传统 ToB 后台（如老版 Ant Design Pro 模板）

### 它们的标志
- 灰白配色、四四方方的卡片、密集表格；
- 侧边栏 + 顶栏的"双导航"结构，菜单层级 ≥ 3 级；
- 操作按钮一律灰底蓝字，确认 + 取消左右对调位置；
- 表单字段密集排列，标签在左、字段在右，移动端体验崩溃；
- 没有"亮 / 暗"两套，只有亮模式。

### 我们绝不做
- ❌ 任何侧边栏（v1 全禁，PM 单点放行才有例外）；
- ❌ 灰底白卡的实色组合；管理端也是毛玻璃 + 红强调；
- ❌ 蓝色或其他第二品牌色；
- ❌ 表单标签在字段左侧；统一在字段上方；
- ❌ 只支持亮模式；暗模式必须 day-1 上线；
- ❌ "确认在左、取消在右" 的反人类排序；我们统一**取消在左 / 确认在右 / 危险按钮始终在右**。

### 我们正在变成它的信号
- review 听到"加个侧边栏放更多模块吧" → 直接打回，请重新做信息架构；
- 出现 ant-design 默认蓝（`#1677ff`）→ 直接打回；
- 表格被塞了 > 10 列 → 必须拆 Drawer 详情 / 移除冗余列。

## 4. 评审红线汇总（一表打印贴墙上）

| 红线 | 触犯即打回 |
|------|----------|
| 出现侧边栏 | ✋ |
| 出现 ≥ 2 种品牌色 | ✋ |
| 任何实色卡片背景 | ✋ |
| 吉祥物 / mascot 当主视觉 | ✋ |
| 任何彩带 / 礼花 / 纸屑 | ✋ |
| 渐变 + 闪烁的进度条 | ✋ |
| 单屏控件 > 8 | ✋ |
| 表格 > 10 列 | ✋ |
| 卡片标题含 emoji | ✋ |
| 暗模式缺失 / 不可用 | ✋ |
| 文案羞辱用户 | ✋ |
| 确认按钮在左 / 取消在右 | ✋ |
