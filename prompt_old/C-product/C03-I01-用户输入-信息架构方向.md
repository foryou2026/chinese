## 3. 业务对象的状态流转（状态机）

> 每个有状态的"东西"列一份。比如：订单、课程、学习进度、审核单。

```
[例：订单状态]
- 状态枚举：待支付 / 已支付 / 已完成 / 已退款 / 已取消
- 待支付 →（支付成功）→ 已支付
- 待支付 →（30 分钟超时 或 用户取消）→ 已取消
- 已支付 →（用户开始第一节课）→ 已完成
- 已支付 →（用户在 7 天内申请退款）→ 已退款
- 终态：已完成 / 已退款 / 已取消
- 不可回退：已支付 不可回到 待支付

[例：课程状态]
- 草稿 → 待审核 → 已上架 → 已下架
- ...
```

> **没有状态的对象不必填**。复杂状态机请尽量描述触发条件、谁能触发、是否可回退。
