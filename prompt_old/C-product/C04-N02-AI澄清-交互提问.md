# 24 · N02 AI 澄清：页面交互提问清单生成模板

> **阶段**：N 页面交互
> **谁产出**：AI（页面交互设计师）
> **落盘**：`docs/A00-meta/questions/N-<page-id>-questions-round<N>.md`

---

## AI 必须遵守

1. **只读**：N 用户输入 + 本 feature C01 R 基线 + docs/C02-permissions/ + 本 feature C03 IA（重点 04-pages.md、03-state-machines.md、01-feature-catalog.md） + docs/B02-ux/ + docs/B03-design/design-system/ + `A00-01`、`A00-03`、`A00-04` + 本模板。
2. **C/D 隔离铁律**：本阶段是产品设计阶段，**严禁**触碰任何后端实现细节——不读 docs/B01-architecture/、不假设接口签名、不假设数据库字段、不假设路由路径，也不向用户提问"这个调什么接口 / 字段叫什么 / URL 怎么写"。这些都是后续开发阶段的事，本阶段产出物里也不得出现。
3. 第一轮回复必为本模板实例化。
4. 阻断级 ≤ 8、优化级 ≤ 15。
5. 任何与 B02 决策、B03 token 冲突的备选不得出现。

---

## 必盘点的 13 维

1. 用户单一目标是否明确（一页只服务 1-2 个核心目标）
2. 区块清单与权重（首屏优先级）
3. 操作清单（OP-ID）与"用户期望系统给什么反馈"是否齐全（**只描述行为与反馈，不写接口**）
4. 状态四件套（默认/加载/空/错误）
5. **状态驱动 UI**：页面如涉及 SM-ID，不同状态下可见元素/可见操作/文案差异是否明确
6. 角色差异（未登录、不同角色、自己 vs 他人、有权 vs 无权）
7. 表单交互（校验时机、错误展示、提交反馈、未保存提示）
8. 关键场景（SC-X）覆盖度（≥ 5 条且含异常）
9. 移动端形态变化
10. 键盘/快捷键/无障碍要求
11. 性能预算（首屏元素、骨架屏、懒加载）
12. 文案语气（与 docs/B02-ux/04-voice-tone.md 一致）
13. SEO / GEO（仅公开页）
