# 06 · A02 AI 澄清：架构提问清单生成模板

> **阶段**：A 技术架构
> **谁产出**：AI（架构师）
> **落盘**：`docs/A00-meta/questions/A-questions-round<N>.md`

---

## AI 必须遵守

1. **只读**：A 用户输入（docs/B01-architecture/_input/preferences.md） + `A00-01`、`A00-03`、`A00-04` + 本模板。
   > B01 是项目级一次性产物，不依赖任何 feature 级的 C01-R 基线。
2. 第一轮回复必为本模板实例化。
3. 阻断级 ≤ 10、优化级 ≤ 20。
4. 每条问题必有：可选项、推荐、推荐理由、影响范围。

---

## 必盘点的 16 维（出题前每维都要摸过）

1. 前后端语言版本与包管理器
2. 仓库结构、模块边界
3. **应用端清单（surface inventory）**与跨端隔离策略（代码库/用户体系/API 路由前缀）
4. 数据库引擎、托管方式（云 / 自托管 / 本地 / Supabase / Neon）、是否引入 BaaS / MCP
5. 数据访问层（ORM / Query builder / 原生 SQL）
6. API 风格（REST / GraphQL / RPC）与版本策略、**多 surface 是否拆路由前缀**
7. **鉴权基础设施（全局技术决策）**：JWT / Session / OAuth / 魔法链接；Token 存储；Refresh 策略；密码哈希算法。具体登录/注册/忘密页面由 feature 级 `<surface>-auth` 走 C 循环，不在本阶段详述
8. 任务队列 / 定时任务
9. 缓存策略 / Redis 用途
10. 日志、监控、追踪
11. 部署形态、CI/CD；**多 surface 是否独立部署**
12. 多语言、时区、货币
13. 测试金字塔（单测/集成/E2E）边界
14. 第三方集成清单与 mock 时机
15. 文件上限、命名、目录约束
16. **AI / LLM 与 AI 工作流框架**（LangGraph / Vercel AI SDK / …）是否引入、如何隔离业务代码与 Agent 代码
