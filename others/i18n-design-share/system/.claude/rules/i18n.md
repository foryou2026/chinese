# i18n Rules

For substantial i18n changes, also read `packages/shared-i18n/README.md`.

## Supported Locales

`zh`, `en`, `vi`, `th`, `id`

## Source Language

English (`en`) is the source language. Write English first, then translate to the other supported locales.

## Adding Text

When adding user-facing text:

1. Add a key to `packages/shared-i18n/src/en.ts`.
2. Add the same key to `zh`, `vi`, `th`, and `id`.
3. Use `t("key.name")` in React components.
4. Do not hardcode visible strings in JSX.

## Translation Quality

- Translate meaning, not word by word.
- Keep button labels short.
- Keep placeholders unchanged.
- Keep punctuation natural for each language.
- If English uses `{{count}}`, every locale must also use `{{count}}`.

## Validation

Run `pnpm --filter @zhiyu/shared-i18n test` after changing locale files.
