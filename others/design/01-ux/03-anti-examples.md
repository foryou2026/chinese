# 反例清单

> 至少 3 个。每个反例说"哪种情况下我们会变成这样、如何防止"。

---

## ANTI-1：Material Design 3 · 强 elevation + ripple 安卓味

- **典型表象**：FAB 圆形悬浮按钮、点击有水波涟漪、卡片 4 级阴影、主色用纯亮蓝 / 紫。
- **若我们走偏会怎样**：青花墨青被替换成亮蓝、按钮投影变重、点击出 ripple——立刻丢失"宋瓷气韵"，沦为通用安卓后台。
- **防御机制**：
  - `06-experience-principles.md` § P3「玻璃为唯一表面语言」；
  - 全局禁用 `material-ripple` 类库；
  - 阴影只允许 `--shadow-1/2/3` 三档，最大 `0 16px 40px rgba(14,31,56,.16)`，不允许新增。

## ANTI-2：Notion 全白 + 灰边 + 满屏 emoji 的"轻博客"

- **典型表象**：背景纯白 `#FFFFFF`、卡片只有 1px 灰边无玻璃、标题前面挂 emoji、文案带"😊 ✨"。
- **若我们走偏会怎样**：失去"温度 + 庄重"，看起来像个人笔记软件，业务严肃感全无。
- **防御机制**：
  - 红线 #1（禁止纯白大面积）；
  - `04-voice-tone.md` 明确禁用 emoji；
  - tokens 不导出 `#FFFFFF` 作为 bg 变量。

## ANTI-3：SaaS 模板站 · Hero 大渐变 + 三栏 features 同质感

- **典型表象**：首页打开就是 100vh 的紫蓝渐变 + 巨大 H1 + 下方"Feature 1 / 2 / 3"卡片三栏；
- **若我们走偏会怎样**："国际化大气磅礴"风视觉 → 与 12306 / 阿里云控制台 / 字节飞书任意一个混淆。
- **防御机制**：
  - Three.js 流体仅在登录 / 欢迎 / 空态 / 错误页允许（`spec/05`）；
  - 业务页 Hero 必须是"信息卡片"而非"营销画面"；
  - 主页布局必须使用 `.glass-strong` 三件套，不允许大渐变背板。

## ANTI-4：传统中医 / 国风装饰陷阱

- **典型表象**：卷轴杆边框、毛笔水墨字标题铺底、印章图作 logo、楷草隶混排、深棕木纹背景。
- **若我们走偏会怎样**：被误认为"古董博物馆 H5 / 中老年养生公众号"。
- **防御机制**：
  - 红线 #2（禁止国风装饰陷阱）；
  - 朱砂仅限印鉴 ≤44px / 激活下划线 2px / 红点 8px；
  - 行楷字仅 3 处允许（品牌名 / Hero H1 / 印鉴）。

## ANTI-5：医院 HIS 蓝 · v3 已废弃

- **典型表象**：纯亮蓝 `#1890FF` 主色 + 浅灰背景 + Element/AntD 默认控件 + 表格斑马纹。
- **若我们走偏会怎样**：彻底丢掉品牌识别，回到"任意一个企业内网"。
- **防御机制**：
  - 主色锚定 `#1B3A5C`，比 Element 蓝深 + 偏冷；
  - 表格禁止斑马纹（`spec/04 §五`）；
  - 禁止直接引入 AntD / Element 默认皮肤，必须经过 token 二次封装。

## 反例 → 原则 映射

| 反例 | 触发的防御原则 |
|------|--------------|
| ANTI-1 | P3（玻璃唯一表面）、P5（动效克制） |
| ANTI-2 | P1（温度但有边界）、P4（文案直说不卖萌） |
| ANTI-3 | P2（信息密度优先）、P6（流体仅气韵不装饰） |
| ANTI-4 | P1、P3、装饰节制铁律 |
| ANTI-5 | P3、token 唯一来源原则 |
